# Node.js 框架与实操五大模块完整详解
## 8. 框架使用 — Express
### 一、框架定位
Express 是 Node.js 最经典、轻量的**Web 服务框架**，基于原生 `http` 模块封装，无内置路由、模板、数据库，极简灵活，生态极其丰富，适合中小型接口服务、后台管理、静态资源服务。
### 核心特性
1. 中间件机制（整个 Express 核心）
2. 极简路由定义，支持 RESTful API
3. 统一处理请求/响应对象
4. 兼容所有 Node 原生模块，扩展插件极多
5. 同步代码友好，不强制异步语法

### 核心对象与常用 API
1. **express()**：创建应用实例 `app`
2. **app.use()**：注册全局中间件、挂载路由、静态资源
3. **app.METHOD(path, handler)**：路由（get/post/put/delete 等）
4. **app.listen(port)**：启动服务监听端口
5. 请求对象 `req`：
   - `req.query`：url 拼接参数
   - `req.body`：post 请求体（需 body-parser 中间件）
   - `req.params`：动态路由参数 `/user/:id`
   - `req.headers`：请求头
6. 响应对象 `res`：
   - `res.send()`：返回任意数据（自动识别类型）
   - `res.json()`：返回 JSON（接口专用）
   - `res.status(200).json()`：自定义状态码
   - `res.sendFile()`：返回本地文件

### 基础完整示例
```js
// 安装：npm i express
const express = require('express')
const app = express()
const port = 3000

// 内置中间件：解析json请求体
app.use(express.json())

// 全局中间件（所有接口都会执行）
app.use((req, res, next) => {
  console.log('请求路径：', req.path)
  next() // 放行，不调用则请求卡住
})

// 普通路由
app.get('/', (req, res) => {
  res.json({ msg: 'express 首页' })
})

// 动态路由
app.get('/user/:id', (req, res) => {
  res.json({ userId: req.params.id, query: req.query })
})

// post 接口
app.post('/login', (req, res) => {
  const { username, password } = req.body
  res.json({ code: 200, data: { username } })
})

// 启动服务
app.listen(port, () => {
  console.log(`服务运行在 http://localhost:${port}`)
})
```
### 中间件分类
1. 内置中间件：`express.json()`、`express.static()`（静态文件托管）
2. 第三方中间件：cors（跨域）、morgan（日志）、multer（文件上传）
3. 自定义中间件：通用鉴权、日志打印、参数校验
4. 错误处理中间件：`(err, req, res, next) => {}` 捕获全局异常

### 优缺点
- 优点：上手简单、生态成熟、企业使用广泛、学习成本低
- 缺点：原生不支持 async/await 异常自动捕获，需要手动封装；回调嵌套易产生地狱；不支持洋葱模型中间件

## 9. 框架使用 — Koa
### 一、框架定位
Koa 由 Express 原班团队开发，**新一代 Web 框架**，基于 ES6+ `async/await` 重构，舍弃回调，采用**洋葱圈中间件模型**，无内置路由、静态、body解析，所有功能依靠第三方中间件。
Koa 分两个大版本：
- Koa1：基于 Generator 函数（已淘汰）
- Koa2：async/await（当前主流）

### 核心特性
1. 洋葱中间件模型（执行顺序：入栈→内层处理→倒序出栈）
2. 原生支持 async/await，异步代码同步写法，异常统一捕获
3. 极简 API，上下文 `ctx` 统一封装 req、res，简化代码
4. 无冗余内置功能，轻量高性能

### 核心对象
1. `new Koa()`：创建应用实例 `app`
2. `app.use(async (ctx, next) => {})`：注册中间件
3. 上下文 `ctx`：
   - `ctx.request` / `ctx.req`：原生请求
   - `ctx.response` / `ctx.res`：原生响应
   - `ctx.query`：url 参数
   - `ctx.params`（需 koa-router）：动态路由
   - `ctx.body`：设置返回数据（替代 res.send/json）
   - `ctx.status`：状态码

### 基础完整示例
```js
// 安装：npm i koa koa-router koa-bodyparser
const Koa = require('koa')
const Router = require('koa-router')
const bodyParser = require('koa-bodyparser')

const app = new Koa()
const router = new Router()
const port = 3000

// 全局中间件：解析请求体
app.use(bodyParser())

// 洋葱中间件演示
app.use(async (ctx, next) => {
  console.log('第一层 进入')
  await next() // 进入下一层中间件
  console.log('第一层 离开')
})

// 路由
router.get('/', async ctx => {
  ctx.body = { msg: 'Koa2 首页' }
})

router.get('/user/:id', async ctx => {
  ctx.body = { id: ctx.params.id, query: ctx.query }
})

router.post('/login', async ctx => {
  const data = ctx.request.body
  ctx.body = { code: 200, data }
})

// 挂载路由
app.use(router.routes(), router.allowedMethods())

app.listen(port, () => {
  console.log(`Koa 服务启动：${port}`)
})
```
### 洋葱中间件执行流程示例
```js
app.use(async (ctx, next) => {
  console.log('1 前置')
  await next()
  console.log('1 后置')
})
app.use(async (ctx, next) => {
  console.log('2 前置')
  await next()
  console.log('2 后置')
})
// 输出顺序：
// 1 前置 → 2 前置 → 接口逻辑 → 2 后置 → 1 后置
```
### Koa vs Express 核心区别
| 对比项     | Express                                | Koa2                             |
| ---------- | -------------------------------------- | -------------------------------- |
| 异步方案   | 回调函数，async 错误需手动捕获         | 原生 async/await，统一异常处理   |
| 中间件模型 | 流水线模型（顺序执行，无后置统一处理） | 洋葱模型，前置后置分层处理       |
| 请求响应   | 分离 req / res 两个对象                | 统一封装 ctx 上下文，更简洁      |
| 内置功能   | 自带路由、静态、简单body解析           | 无内置能力，全部依赖第三方中间件 |
| 错误处理   | 异步异常会直接崩溃进程                 | 上层中间件可全局捕获所有异步错误 |

### 优缺点
- 优点：异步代码优雅、中间件灵活、性能更好、适合大型项目分层开发
- 缺点：配套中间件需要单独安装，新手入门需要理解洋葱模型

## 10. Node.js 使用 — 文件和目录（fs 模块）
### 模块分类
Node 内置 `fs` 文件系统模块，三套 API：
1. 同步 API（带 Sync，会阻塞事件循环，仅初始化使用）
2. 回调式异步 API（旧写法）
3. Promise API `fs/promises`（推荐，配合 async/await）

配套流模块：`fs.createReadStream` / `fs.createWriteStream` 处理**大文件**，避免一次性加载全部内容占用内存。

### 一、目录操作
```js
const fs = require('fs/promises')
const fsSync = require('fs')
const path = require('path')

// 1. 创建目录（递归创建多层文件夹 {recursive:true}）
await fs.mkdir('./upload/avatar', { recursive: true })

// 2. 读取目录，返回目录内所有文件名
const files = await fs.readdir('./upload')

// 3. 判断路径是否存在
try {
  await fs.access('./upload')
  console.log('目录存在')
} catch {
  console.log('目录不存在')
}

// 4. 删除空目录；非空目录需先删除内部文件
await fs.rmdir('./test')
// 删除文件夹（包含内部所有文件，node14+）
await fs.rm('./test-folder', { recursive: true, force: true })
```

### 二、文件读写操作
```js
// 读取完整文件（小文件适用）
const content = await fs.readFile('./config.json', 'utf8')

// 写入文件，覆盖原有内容
await fs.writeFile('./log.txt', '日志内容\n')

// 追加写入（不覆盖，末尾新增）
await fs.appendFile('./log.txt', '新增一行日志\n')

// 删除文件
await fs.unlink('./log.txt')

// 获取文件信息（大小、创建时间、是否文件/目录）
const stat = await fs.stat('./index.js')
console.log(stat.isFile(), stat.size)
```

### 三、大文件流式处理（核心）
一次性读取大文件会占用巨量内存，必须使用流：
```js
const rs = fs.createReadStream('big-video.mp4')
const ws = fs.createWriteStream('copy-video.mp4')
// 管道自动流转，自动处理背压
rs.pipe(ws)

// 监听流事件
rs.on('end', () => console.log('文件拷贝完成'))
rs.on('error', err => console.error('读取失败', err))
```

### 四、路径辅助模块 path
单独处理跨平台路径问题（Windows / Linux / Mac 分隔符不同）
```js
const path = require('path')
console.log(path.join(__dirname, 'upload', '1.jpg')) // 拼接绝对路径
console.log(path.extname('a.png')) // 获取后缀 .png
console.log(path.basename('/test/1.jpg')) // 1.jpg
```

### 使用规范
1. 业务接口禁止使用 `xxxSync` 同步方法，阻塞服务
2. 100MB 以上文件必须使用流读写，禁止 readFile
3. 操作文件前使用 `path` 拼接绝对路径，避免相对路径报错
4. 文件操作统一加 try/catch 捕获读写异常

## 11. Node.js 使用 — 应用程序开发完整流程
### 一、项目整体架构分层
通用 Node 后端分层（Express/Koa 通用）
1. **路由层 router**：接收客户端请求，分发接口
2. **中间件层 middleware**：跨接口通用逻辑（跨域、鉴权、日志、参数校验）
3. **控制器层 controller**：接口业务逻辑，处理参数、调用服务
4. **服务层 service**：纯业务处理、数据库操作、第三方调用（解耦控制器）
5. **工具层 utils**：加密、文件、日期、请求封装通用函数
6. **配置层 config**：环境变量、数据库地址、密钥、端口
7. **静态资源 / 上传目录**：图片、附件存放

### 二、完整开发步骤
1. **项目初始化**
```bash
mkdir node-server && cd node-server
npm init -y
# 安装框架依赖
npm i koa koa-router koa-bodyparser cors
# 开发依赖（热更新）
npm i -D nodemon
```
配置 package.json 启动脚本
```json
"scripts": {
  "dev": "nodemon app.js",
  "start": "node app.js"
}
```

2. **环境区分**
使用 `dotenv` 加载 `.env` 文件区分开发/生产环境，存放数据库账号、密钥等隐私配置
```js
require('dotenv').config()
const port = process.env.PORT || 3000
```

3. **全局异常捕获**
- 框架中间件捕获接口异常
- 顶层捕获未捕获异步错误 `process.on('unhandledRejection')`
- 捕获同步崩溃 `process.on('uncaughtException')`

4. **日志系统**
使用 winston 记录访问日志、错误日志，按文件分割，方便线上排查问题

5. **数据库集成**
- MySQL：mysql2 / sequelize ORM
- MongoDB：mongoose
- Redis：ioredis 缓存、会话存储

6. **上线部署**
- 进程守护：pm2 管理 Node 应用，自动重启、负载均衡、日志聚合
- 反向代理：Nginx 做端口转发、静态资源、SSL 证书、限流

### 标准项目目录结构
```
node-server/
├── app.js          # 入口启动文件
├── .env            # 环境配置
├── config/         # 配置文件
├── router/         # 所有路由
├── controller/     # 接口控制器
├── service/        # 业务服务
├── middleware/     # 自定义中间件
├── utils/          # 工具函数（crypto、文件工具）
├── upload/         # 文件上传目录
├── logs/           # 日志文件夹
└── package.json
```

### 典型应用场景
1. RESTful 后端接口服务（管理后台、小程序/APP服务端）
2. 静态资源服务器、图片文件上传服务
3. 定时任务脚本（配合 node-schedule）
4. 爬虫、中间层代理服务
5. WebSocket 实时通讯（聊天室、消息推送）

## 12. Node.js 使用 — Crypto 加密模块
`crypto` 为 Node.js**内置模块**，无需安装，提供哈希、对称加密、非对称RSA、HMAC签名、证书全套能力，用于密码存储、接口签名、数据传输加密。

### 1. Hash 单向哈希（不可逆，用于密码存储、文件校验）
支持算法：md5、sha256、sha512
```js
const crypto = require('crypto')
// 创建sha256哈希
const hash = crypto.createHash('sha256')
hash.update('原始明文密码123456')
// 输出十六进制字符串
const result = hash.digest('hex')
console.log(result)
```
> 注意：单纯 sha256 不安全，生产密码推荐使用 bcrypt / crypto.pbkdf2（加盐哈希）

### 2. HMAC 带密钥哈希（接口签名，防篡改）
双方约定密钥，相同数据+密钥才能算出相同摘要，用于接口请求校验，防止参数被篡改
```js
const hmac = crypto.createHmac('sha256', '自定义密钥secret123')
hmac.update('请求参数data')
const sign = hmac.digest('base64')
console.log('接口签名：', sign)
```

### 3. pbkdf2 加盐密码加密（生产密码存储标准）
随机加盐，暴力破解难度极高，用户密码存储首选
```js
// 加密
crypto.pbkdf2('123456', crypto.randomBytes(16), 10000, 64, 'sha256', (err, key) => {
  console.log(key.toString('hex'))
})
// async/await 写法
const util = require('util')
const pbkdf2 = util.promisify(crypto.pbkdf2)
const salt = crypto.randomBytes(16)
const key = await pbkdf2('123456', salt, 10000, 64, 'sha256')
```

### 4. AES 对称加密（加密传输数据，加解密同一密钥）
Cipheriv / Decipheriv 标准AES（需要密钥key、偏移向量iv）
```js
// 加密
function aesEncrypt(text, key, iv) {
  const cipher = crypto.createCipheriv('aes-128-cbc', key, iv)
  let res = cipher.update(text, 'utf8', 'hex')
  res += cipher.final('hex')
  return res
}
// 解密
function aesDecrypt(cipherText, key, iv) {
  const decipher = crypto.createDecipheriv('aes-128-cbc', key, iv)
  let res = decipher.update(cipherText, 'hex', 'utf8')
  res += decipher.final('utf8')
  return res
}
```

### 5. RSA 非对称加密（公钥加密、私钥解密；私钥签名、公钥验签）
适用场景：支付签名、敏感信息传输
- 公钥 publicKey：对外公开，加密数据
- 私钥 privateKey：本地保存，解密、生成签名

```js
// 私钥签名
const sign = crypto.createSign('sha256')
sign.update('传输数据')
const signStr = sign.sign(privateKey, 'base64')

// 公钥验签
const verify = crypto.createVerify('sha256')
verify.update('传输数据')
const isValid = verify.verify(publicKey, signStr, 'base64')
console.log('签名是否合法：', isValid)
```

### 安全使用注意事项
1. 同步加密方法会阻塞事件循环，大量加密运算放到 worker_threads 线程执行
2. 密钥、私钥禁止硬编码写代码，统一放入环境变量 .env
3. 禁止使用 md5 存储密码，无加盐极易彩虹表破解
4. AES 加密必须使用随机 iv，不能固定向量
5. 线上生产优先使用高安全算法 sha256/sha512，废弃 md5、des

### 业务使用场景
1. 用户登录密码加密存储
2. 前后端接口请求签名校验，防篡改、伪造请求
3. 敏感报文AES加密传输
4. 文件完整性校验（上传文件计算hash，校验是否损坏）
5. 支付、第三方对接 RSA 签名验签
