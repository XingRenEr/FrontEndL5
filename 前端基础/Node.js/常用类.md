# Node.js 七大核心常用类完整详解
Node.js 内置大量原生模块，按用途划分为加密、全局、平台、进程、网络、线程、文件七大类，下面分模块讲解**核心类、API、使用场景、代码示例、注意事项**。

## 一、加密类 crypto（内置加密模块）
### 核心作用
实现哈希、对称/非对称加密、签名、证书、编码，用于密码存储、接口签名、数据脱敏。
### 核心类
1. **Hash**：哈希摘要（md5、sha256、sha512）
2. **Hmac**：带密钥哈希，防篡改签名
3. **Cipher / Decipher**：对称加密/解密（aes、des）
4. **Sign / Verify**：非对称签名验签（rsa、ecdsa）
5. **Cipheriv/Decipheriv**：带向量iv的标准对称加密（生产推荐）
6. **KeyObject**：统一管理密钥对象
7. **Certificate**：x509证书解析

### 示例
```js
const crypto = require('crypto')
// Hash 密码摘要
const hash = crypto.createHash('sha256')
hash.update('123456')
console.log(hash.digest('hex'))

// Hmac 接口签名
const hmac = crypto.createHmac('sha256', 'secret-key')
hmac.update('data')
console.log(hmac.digest('base64'))
```
### 场景
用户密码加密、API 请求签名、文件校验、数据传输加密。

## 二、全局类 global / 全局内置对象
### 核心作用
无需 require，Node 运行时直接访问的顶层对象，浏览器 window 对等物。
### 核心全局类/对象
1. **Buffer**：二进制缓冲区（Node 独有，处理字节流）
2. **process**：进程全局对象（归进程类，此处区分全局顶层）
3. **console**：日志输出类
4. **setTimeout / setInterval / setImmediate**：定时器类
5. **URL / URLSearchParams**：地址解析全局类
6. **TextEncoder / TextDecoder**：编解码文本
7. __dirname / __filename：文件路径全局变量
8. globalThis：跨环境统一全局入口

### 重点：Buffer 核心类
Node 处理二进制核心，字符串、文件、网络数据底层都是 Buffer
```js
const buf = Buffer.from('hello node')
console.log(buf.toString('utf8'))
```
### 场景
二进制读写、编码转换、日志打印、延时任务、URL 解析。

## 三、平台相关类 os 操作系统模块
### 核心作用
获取操作系统底层信息，区分 Windows / Linux / macOS，适配跨平台逻辑。
### 核心类/API
1. **os.Constants**：系统常量（信号、错误码）
2. 内置工具函数（底层封装平台结构体）
    - os.cpus()：CPU 核心信息
    - os.totalmem() / freemem()：内存
    - os.platform()：平台标识 win32/linux/darwin
    - os.homedir()：用户主目录
    - os.networkInterfaces()：网卡信息

### 示例
```js
const os = require('os')
console.log('CPU核心：', os.cpus().length)
console.log('操作系统：', os.platform())
console.log('空闲内存：', os.freemem() / 1024 / 1024)
```
### 场景
服务监控、跨平台路径适配、硬件资源检测、获取本机IP。

## 四、进程类 process + child_process 子进程
分为**主进程 process（全局）** 和**子进程 ChildProcess** 两大块
### 1. process 主进程类（全局）
#### 核心能力
- 进程环境：env 环境变量、argv 启动参数
- 事件监听：exit、uncaughtException、SIGINT（ctrl+c）
- 资源控制：内存、CPU、标准流 stdin/stdout/stderr
- 操作：process.kill()、chdir 切换目录

### 2. child_process 子进程模块核心类 ChildProcess
#### 四大创建方式
1. spawn：流式输出，适合大量日志（shell 命令）
2. exec：缓存输出，有缓冲区上限
3. execFile：安全执行可执行文件，无shell注入风险
4. fork：spawn 封装，专门创建 Node 子进程，内置 IPC 通信

#### ChildProcess 实例核心事件
- message：父子进程IPC通信
- exit：子进程退出
- error：启动失败
- stdout/stderr：标准输出流

### 示例
```js
const { fork } = require('child_process')
// 创建子进程
const child = fork('./sub.js')
// 进程通信
child.send({ type: 'ping' })
child.on('message', msg => console.log('子进程消息', msg))
```
### 场景
执行 shell 脚本、多进程任务拆分、独立运行耗时任务、程序启停管理。

## 五、网络类 net / http / https / dns
### 5.1 net 底层TCP网络类（基础）
核心类：
1. **net.Server**：TCP服务端
2. **net.Socket**：TCP客户端/连接套接字（双向流）
提供原始TCP长连接，WebSocket、HTTP底层依赖net。
```js
const net = require('net')
const server = net.createServer(sock => {
  sock.write('tcp hello')
  sock.on('data', d => console.log(d.toString()))
})
server.listen(3000)
```

### 5.2 http 应用层HTTP模块
核心类：
1. **http.Server**：HTTP服务
2. **http.IncomingMessage**：请求对象 req
3. **http.ServerResponse**：响应对象 res
4. **http.ClientRequest**：HTTP客户端请求

### 5.3 https TLS加密HTTP
基于 tls 模块封装，类结构和 http 完全一致，增加证书配置。

### 5.4 dns 域名解析类
解析域名、反向解析IP，底层封装操作系统DNS库。
### 场景
TCP长连接服务、HTTP接口服务、爬虫、代理服务器、后端接口、HTTPS网站。

## 六、线程类 worker_threads 工作线程
### 背景
Node.js 主线程单线程，CPU密集计算会阻塞事件循环，worker_threads 创建**独立线程**，共享内存，不阻塞主线程。
### 核心类
1. **Worker**：创建工作线程实例
2. **MessageChannel / MessagePort**：线程间通信通道
3. **SharedArrayBuffer**：多线程共享内存（大数据高性能传输）
4. **Atomics**：共享内存原子操作，解决线程并发竞争

### 示例
```js
const { Worker } = require('worker_threads')
const worker = new Worker('./cpu-task.js')
worker.postMessage({ num: 99999 })
worker.on('message', res => console.log('计算结果', res))
```
### 区分：进程 vs 线程
- 子进程：独立V8实例，开销大，IPC通信
- Worker线程：共享进程V8，轻量，适合密集计算
### 场景
大文件解析、加解密运算、图像处理、大数据循环计算，避免阻塞接口。

## 七、文件类 fs 文件系统模块
### 两套API
1. 回调版 fs.*(path, callback)
2. Promise 版 fs/promises（async/await，推荐）
### 核心类
1. **fs.Stats**：文件元信息（大小、创建时间、是否目录）
2. **fs.ReadStream**：文件可读流（大文件读取）
3. **fs.WriteStream**：文件可写流（日志写入）
4. **fs.Dir / fs.Dirent**：目录遍历、读取目录条目

### 核心能力
- 文件：readFile / writeFile / appendFile
- 目录：mkdir / readdir / rmdir
- 硬链接/软链接：link / symlink
- 文件监控：fs.watch 文件变动监听

### 流式读写示例（处理大文件）
```js
const fs = require('fs')
const rs = fs.createReadStream('big-file.txt')
const ws = fs.createWriteStream('output.txt')
rs.pipe(ws) // 流式拷贝，不占用大量内存
```
### 场景
日志持久化、静态文件服务、配置文件读写、文件上传下载、本地资源管理。

# 七大类对比总结
| 分类   | 核心模块               | 解决核心问题              | 阻塞风险                         |
| ------ | ---------------------- | ------------------------- | -------------------------------- |
| 加密类 | crypto                 | 数据加密、签名、哈希      | 同步加密CPU阻塞，推荐异步/worker |
| 全局类 | global、Buffer         | 通用顶层工具、二进制处理  | Buffer操作轻量无阻塞             |
| 平台类 | os                     | 获取操作系统硬件/平台信息 | 同步读取，几乎无阻塞             |
| 进程类 | process、child_process | 多进程、执行外部命令      | 子进程独立，不阻塞主线程         |
| 网络类 | net/http/https         | TCP/HTTP网络通信          | 基于事件循环，异步非阻塞         |
| 线程类 | worker_threads         | CPU密集计算防阻塞         | 线程隔离，不阻塞主线程           |
| 文件类 | fs                     | 本地文件/目录读写         | 同步fs阻塞，优先promise/流       |
