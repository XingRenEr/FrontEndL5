# TypeScript 面向对象四大核心：封装、继承、多态、面向对象编程详解
TypeScript 在 ES6 Class 基础上，通过**类型、访问修饰符、抽象类、接口**完整落地标准面向对象三大特性：封装、继承、多态，同时规范面向对象编程范式。

## 一、封装（Encapsulation）
### 1. 核心概念
将**数据（属性）**和**操作数据的方法**捆绑在类内部，对外只暴露安全访问入口，隐藏内部私有细节，避免外部随意篡改内部状态。
TS 通过三类访问修饰符实现权限隔离，是封装的核心手段：
| 修饰符      | 作用范围                                  |
| ----------- | ----------------------------------------- |
| `public`    | 默认，类内部、子类、实例均可访问          |
| `private`   | 仅当前类内部可访问，子类/实例都无法读写   |
| `protected` | 当前类 + 子类内部可访问，外部实例禁止访问 |

### 2. 基础封装示例
```typescript
class User {
  // 私有属性：外部无法直接修改，实现数据隐藏
  private _password: string;
  public name: string;

  constructor(name: string, pwd: string) {
    this.name = name;
    this._password = pwd;
  }

  // 对外暴露只读获取密码方法（安全访问入口）
  getPassword(): string {
    return "******"; // 不返回真实密码，保护数据
  }

  // 受控修改密码，增加校验逻辑
  setPassword(newPwd: string): void {
    if (newPwd.length < 6) {
      throw new Error("密码长度不能小于6位");
    }
    this._password = newPwd;
  }
}

const u = new User("张三", "123456");
console.log(u.name);
// u._password 直接访问私有属性，编译报错
console.log(u.getPassword());
u.setPassword("654321");
// u.setPassword("123") 抛出错误，拦截非法数据
```

### 3. 存取器 get/set（封装简化写法）
替代单独的 get/set 方法，语法更简洁：
```typescript
class User {
  private _age: number = 0;

  get age(): number {
    return this._age;
  }

  set age(val: number) {
    if (val < 0 || val > 150) throw new Error("年龄非法");
    this._age = val;
  }
}
const user = new User();
user.age = 20; // 自动调用set
console.log(user.age); // 自动调用get
```

### 4. 封装价值
1. 数据保护：禁止外部随意篡改内部字段；
2. 统一校验：所有修改逻辑集中在类内部；
3. 低耦合：内部实现变更，外部调用代码无需修改。

## 二、继承（Inheritance）
### 1. 核心概念
允许一个类（**子类/派生类**）复用另一个类（**父类/基类**）的属性与方法，子类可拓展自身独有逻辑，代码复用，减少重复代码。
TS 关键字：`extends` 实现继承，`super()` 调用父类构造函数。

### 2. 基础继承示例
```typescript
// 父类
class Animal {
  protected name: string;
  constructor(name: string) {
    this.name = name;
  }
  eat(): void {
    console.log(`${this.name}吃东西`);
  }
}

// 子类 Dog 继承 Animal
class Dog extends Animal {
  constructor(name: string) {
    // 必须先调用父类构造器
    super(name);
  }
  // 子类拓展独有方法
  bark(): void {
    console.log(`${this.name}汪汪叫`);
  }
}

const dog = new Dog("柯基");
dog.eat(); // 继承父类方法
dog.bark(); // 子类自有方法
```

### 3. 方法重写 override
子类可以重新实现父类同名方法，覆盖原有逻辑；TS 4.3+ 支持 `override` 关键字标记重写，规范代码：
```typescript
class Cat extends Animal {
  constructor(name: string) {
    super(name);
  }
  // 重写父类eat方法
  override eat(): void {
    console.log(`${this.name}吃鱼`);
  }
}
const cat = new Cat("橘猫");
cat.eat(); // 输出：橘猫吃鱼
```

### 4. protected 继承特性
`protected` 属性仅子类可见，外部实例无法访问，专门为继承设计：
```typescript
class Dog extends Animal {
  test() {
    console.log(this.name); // protected 子类可读取
  }
}
const d = new Dog("柴犬");
// d.name 外部访问报错
```

### 5. 继承限制
TS/JS 仅支持**单继承**，一个类只能 `extends` 一个父类；多规范通过 `implements` 实现接口达成。

## 三、多态（Polymorphism）
### 1. 核心概念
同一个方法，不同子类对象执行时表现出不同行为。
实现多态两大前提：
1. 存在父类与子类继承关系；
2. 子类重写父类同名方法；
3. 使用父类类型接收子类实例。

### 2. 多态完整示例
```typescript
// 父类
abstract class Animal {
  abstract speak(): void;
}

// 子类1
class Dog extends Animal {
  speak(): void {
    console.log("汪汪汪");
  }
}
// 子类2
class Cat extends Animal {
  speak(): void {
    console.log("喵喵喵");
  }
}
// 子类3
class Bird extends Animal {
  speak(): void {
    console.log("叽叽叽");
  }
}

// 多态函数：参数为父类类型，可以接收任意子类实例
function animalTalk(animal: Animal) {
  animal.speak(); // 同一方法，不同对象不同表现
}

animalTalk(new Dog());  // 汪汪汪
animalTalk(new Cat());  // 喵喵喵
animalTalk(new Bird()); // 叽叽叽
```

### 3. 基于接口实现多态（更常用）
接口定义统一行为规范，多个类实现接口，同样达成多态效果，工程中更推荐：
```typescript
interface ISpeak {
  speak(): void;
}

class Dog implements ISpeak {
  speak() { console.log("汪") }
}
class Human implements ISpeak {
  speak() { console.log("你好") }
}

function say(obj: ISpeak) {
  obj.speak();
}
say(new Dog());
say(new Human());
```

### 4. 多态价值
面向接口/父类编程，代码解耦，新增子类无需修改原有调用逻辑，符合开闭原则。

## 四、面向对象编程（OOP 完整范式）
### 1. 什么是面向对象编程
和面向过程（按步骤写函数）对立，OOP 核心思想：**把程序看作一组对象的集合**。
- 对象：拥有属性（数据）+ 方法（行为）；
- 类：对象的模板，批量创建同类对象；
三大支柱：封装、继承、多态；
配套能力：抽象类、接口、泛型、访问修饰符。

### 2. OOP 四大特性完整总结
1. **封装**：藏内部，露接口，管控数据；
2. **继承**：复用父类代码，拓展子类能力；
3. **多态**：统一调用入口，不同对象不同实现；
4. **抽象**（隐含基础）：抽象类/接口提取通用规范，只定义行为不写实现。

### 3. TS 面向对象完整综合案例（融合三大特性）
```typescript
// 抽象（抽象类，定义通用规范）
abstract class Goods {
  constructor(
    private _name: string,
    protected price: number
  ) {}

  // 封装：读取名称
  get name() {
    return this._name;
  }
  // 抽象方法，子类必须实现（多态基础）
  abstract showInfo(): void;
}

// 子类1 继承
class Phone extends Goods {
  constructor(name: string, price: number) {
    super(name, price);
  }
  // 重写实现抽象方法，多态行为1
  override showInfo(): void {
    console.log(`手机：${this.name}，售价${this.price}元`);
  }
}

// 子类2 继承
class Book extends Goods {
  constructor(name: string, price: number) {
    super(name, price);
  }
  // 重写实现抽象方法，多态行为2
  override showInfo(): void {
    console.log(`书籍：${this.name}，售价${this.price}元`);
  }
}

// 多态调用函数
function printGoods(goods: Goods) {
  goods.showInfo();
}

// 实例
const iphone = new Phone("iPhone 16", 5999);
const tsBook = new Book("TypeScript入门", 59);

printGoods(iphone);
printGoods(tsBook);
```

### 4. 面向对象编程适用场景
1. 业务模型复杂，存在大量实体（用户、订单、商品）；
2. 代码需要大量复用、分层拓展；
3. 框架开发（Vue/React 类组件、后端 Node 服务）；

### 5. OOP 对比面向过程
- 面向过程：关注**怎么做**，拆分步骤函数，数据与逻辑分离；
- 面向对象：关注**谁来做**，拆分对象，数据和行为绑定，易维护、易扩展。

### 6. TS OOP 独有优势（相比原生JS）
1. 访问修饰符强制封装，编译阶段拦截非法访问；
2. 抽象类、接口强制规范子类实现，约束多态；
3. 静态类型校验，类、继承、多态全程类型提示，减少运行报错；
4. override 关键字规范重写，避免无意覆盖父类方法。
