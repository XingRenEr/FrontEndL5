# TypeScript 完整基础语法详解
TypeScript 是 JavaScript 的超集，核心是**静态类型系统**，在 JS 基础上新增类型校验、类、接口、泛型、装饰器等工程化能力，下面按你列出的八大模块完整拆解。

## 一、静态类型（TS 核心根基）
JS 是动态弱类型，运行时才校验类型；TS 静态类型在**编译阶段**做类型检查，提前规避类型错误。
### 1. 基础类型注解
```typescript
// 基础原始类型
let str: string = "ts";
let num: number = 10;
let flag: boolean = true;
let n: null = null;
let u: undefined = undefined;
// 数组
let arr: number[] = [1,2,3];
let arr2: Array<string> = ["a"];
// 对象
let obj: { name: string; age: number } = { name: "张三", age: 18 };
// 元组（固定长度、固定类型数组）
let tuple: [string, number] = ["李四", 20];
// void 无返回值
function log(): void { console.log(1) }
// any 任意类型（放弃类型校验，慎用）
let data: any = "abc";
data = 123;
// unknown 未知类型（安全的any，必须类型缩小才能使用）
let val: unknown = "test";
if(typeof val === "string") val.toUpperCase();
// never 永远不会有返回（死循环/抛出错误）
function err(): never { throw new Error("报错") }
```
### 2. 类型推导
TS 会自动推导类型，无需手动注解：
```typescript
let count = 10; // 自动推导为 number
count = "10"; // 编译报错
```
### 3. 联合/交叉类型（基础静态类型拓展）
```typescript
// 联合：多种类型其一
let id: string | number = 100;
id = "s100";
// 交叉：同时满足多个类型
type User = { name: string };
type Age = { age: number };
type Person = User & Age;
const p: Person = { name: "小明", age: 16 };
```

## 二、类、接口、枚举
### 1. 类 Class（面向对象封装）
TS 在 ES6 class 基础上增加**访问修饰符、类型、抽象类**
#### 访问修饰符
- `public`：默认，任意位置访问
- `private`：仅类内部访问
- `protected`：类内部 + 子类可访问
```typescript
class Animal {
  public name: string;
  private weight: number;
  protected age: number;

  constructor(name: string, weight: number, age: number) {
    this.name = name;
    this.weight = weight;
    this.age = age;
  }

  // 实例方法
  run(): void {
    console.log(`${this.name}奔跑`);
  }
}

// 继承 extends
class Dog extends Animal {
  constructor(name: string, weight: number, age: number) {
    super(name, weight, age);
  }
  bark() {
    console.log(this.age); // protected 子类可用
    // this.weight 私有，报错
  }
}
```
#### 抽象类 abstract
不能实例化，用于定义通用模板，必须被子类实现抽象方法
```typescript
abstract class Shape {
  abstract getArea(): number; // 抽象方法无实现
}
class Circle extends Shape {
  r: number;
  constructor(r: number) { super(); this.r = r }
  getArea() { return Math.PI * this.r ** 2 }
}
```

### 2. 接口 Interface
用于**约束对象结构、函数、类**，只定义规范，无实现代码
#### 1）约束对象
```typescript
interface IUser {
  name: string;
  age?: number; // 可选属性
  readonly id: number; // 只读属性
  [prop: string]: any; // 任意扩展属性
}
const user: IUser = { id: 1, name: "小红" };
```
#### 2）约束类实现 `implements`
```typescript
interface IPlay {
  play(): void;
}
class Cat implements IPlay {
  play() { console.log("玩毛线") }
}
```
#### 3）接口继承 extends
```typescript
interface Person { name: string }
interface Student extends Person { studentId: number }
```
#### interface vs type
- interface：可重复声明自动合并，适合描述对象/类
- type：支持联合、交叉、基础类型别名，不能重复合并

### 3. 枚举 Enum
固定一组常量集合，提升代码可读性，分**数字枚举、字符串枚举**
```typescript
// 数字枚举（默认从0自增）
enum Status {
  Off, // 0
  On,  // 1
  Loading = 5 // 手动赋值，后续顺延
}
console.log(Status.Off); // 0
console.log(Status[0]); // "Off" 反向映射

// 字符串枚举（无反向映射，推荐）
enum Color {
  Red = "RED",
  Blue = "BLUE"
}

// 常量枚举 const enum，编译后直接替换，减少代码体积
const enum Dir { Left, Right }
let d = Dir.Left; // 编译后 let d = 0;
```

## 三、函数
TS 给函数增加**参数类型、返回值类型、可选参数、默认参数、剩余参数、函数类型**
### 1. 基础函数类型标注
```typescript
// 完整标注
function add(a: number, b: number): number {
  return a + b;
}
// 箭头函数
const sub = (a: number, b: number): number => a - b;
```
### 2. 可选 / 默认参数
```typescript
// 可选参数 ?
function log(name: string, msg?: string) {}
// 默认参数
function createUser(name: string, age: number = 18) {}
```
### 3. 剩余参数
```typescript
function sum(base: number, ...nums: number[]): number {
  return nums.reduce((s, i) => s + i, base);
}
```
### 4. 函数类型别名
```typescript
type CalcFn = (x: number, y: number) => number;
const mul: CalcFn = (a, b) => a * b;
```
### 5. 函数重载
同一函数名，多套参数类型匹配，实现多态
```typescript
// 重载声明
function getInfo(id: number): string;
function getInfo(name: string): number;
// 统一实现
function getInfo(arg: string | number): string | number {
  if(typeof arg === "number") return `id:${arg}`;
  return arg.length;
}
```
### 6. this 类型标注
```typescript
interface Person {
  name: string;
  say: (this: Person) => void;
}
const p: Person = {
  name: "老王",
  say() { console.log(this.name) }
}
```

## 四、泛型 Generic
解决**类型复用**，不固定类型，使用时传入具体类型，避免 any 丢失类型提示。
### 1. 基础泛型
```typescript
// T 类型占位符
function getVal<T>(arg: T): T {
  return arg;
}
getVal<string>("test");
getVal(100); // 自动推导 T=number
```
### 2. 泛型约束 extends
限制传入类型必须具备某些属性
```typescript
interface HasLen { length: number }
// T 必须包含 length 属性
function getLength<T extends HasLen>(arg: T): number {
  return arg.length;
}
getLength("abc");
getLength([1,2]);
```
### 3. 泛型接口、泛型类
```typescript
// 泛型接口
interface IData<T> {
  data: T;
}
const res: IData<number[]> = { data: [1,2] };

// 泛型类
class Store<T> {
  list: T[] = [];
  push(item: T) { this.list.push(item) }
}
const numStore = new Store<number>();
numStore.push(1);
```
### 4. 多泛型参数
```typescript
function pair<T, K>(a: T, b: K): [T, K] {
  return [a, b];
}
pair<string, number>("id", 10);
```

## 五、高级类型
### 1. 类型别名 type
```typescript
type StrNum = string | number;
type ObjType = { x: number; y: number };
```
### 2. 类型推断 & 类型断言
```typescript
// 类型断言两种写法
let val: unknown = "hello";
let len1 = (val as string).length;
let len2 = (<string>val).length;
```
### 3. 映射类型（批量改造类型）
内置工具类型本质是映射类型
```typescript
type User = { name: string; age: number };
// Readonly：全部只读
type ReadUser = Readonly<User>;
// Partial：全部可选
type PartialUser = Partial<User>;
// Required：全部必填
type ReqUser = Required<PartialUser>;
// Pick：挑选部分属性
type PickUser = Pick<User, "name">;
// Omit：剔除部分属性
type OmitUser = Omit<User, "age">;
```
### 4. 条件类型
`T extends U ? X : Y` 根据类型判断分支
```typescript
type IsStr<T> = T extends string ? true : false;
type A = IsStr<"abc">; // true
type B = IsStr<123>; // false
```
### 5. 索引类型
```typescript
type User = { name: string; age: number };
type UserKey = keyof User; // "name" | "age"

function getProp<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
const u: User = { name: "aa", age: 10 };
getProp(u, "name");
```
### 6. 模板字面量类型
操作字符串类型
```typescript
type EventName = `on${Capitalize<string>}`;
type ClickEvt = `onClick`;
```

## 六、模块和命名空间
用于代码隔离、避免全局变量污染，TS 两套模块化方案：**ES Module（推荐）、namespace 命名空间**
### 1. ES Module（现代项目标准）
文件天然模块，使用 `import / export`
```typescript
// user.ts 导出
export interface IUser { name: string }
export const createUser = () => ({ name: "xx" });
export default class User {}

// index.ts 导入
import User, { IUser, createUser } from "./user";
```
编译配置 `tsconfig.json` 需设置 `module: ESNext`

### 2. 命名空间 namespace（旧浏览器/全局脚本使用）
用于单文件内隔离代码，用 `namespace + export` 暴露内部内容
```typescript
namespace Utils {
  export function log(msg: string) {
    console.log(msg);
  }
  const secret = "私有变量"; // 外部不可访问
}
Utils.log("测试");
```
多文件命名空间合并：通过 `/// <reference path="xxx.ts"/>` 引入文件

### 区别
- ES Module：工程化、打包友好、前端主流
- namespace：无打包工具、多全局脚本场景，新项目不推荐

## 七、装饰器 Decorator
一种语法糖，**运行时拦截类/方法/属性**，基于 ES7 提案，需要开启 `tsconfig.json`：
```json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```
### 五大装饰器类型（执行顺序：参数→方法→访问器→属性→类）
1. **类装饰器**：修饰整个类
```typescript
function Logger(target: Function) {
  console.log("创建类", target.name);
}
@Logger
class Service {}
```
2. **方法装饰器**：修饰类方法
```typescript
function LogMethod(target: any, key: string, desc: PropertyDescriptor) {
  const oldFn = desc.value;
  desc.value = function(...args: any[]) {
    console.log("调用方法", key);
    return oldFn.apply(this, args);
  }
}
class User {
  @LogMethod
  say() {}
}
```
3. **属性装饰器**
4. **访问器装饰器（get/set）**
5. **参数装饰器**

### 装饰器工厂（传参装饰器）
```typescript
function Log(text: string) {
  return function(target: Function) {
    console.log(text, target);
  }
}
@Log("用户服务")
class UserService {}
```
常用场景：Vue/Angular、依赖注入、路由拦截、日志埋点。

## 八、声明文件 .d.ts
TS 识别第三方 JS 库、全局变量、模块类型的文件，后缀 `.d.ts`，仅类型定义，无业务代码。
### 1. 作用
- 给纯 JS 库提供类型提示（如 lodash、jquery）
- 自定义全局变量、全局函数
- 自定义模块类型（处理无类型包）

### 2. 基础语法
#### 1）声明全局变量
```typescript
// global.d.ts
declare const APP_VERSION: string;
declare function toast(msg: string): void;
```
全局任意文件可直接使用 `APP_VERSION`、`toast()`

#### 2）声明模块（解决导入无类型包报错）
```typescript
// 处理图片/静态资源导入
declare module "*.png" {
  const src: string;
  export default src;
}
declare module "*.vue" {
  import { DefineComponent } from "vue";
  const comp: DefineComponent;
  export default comp;
}
```

#### 3）声明类型/接口（全局可用）
```typescript
declare interface GlobalUser {
  id: number;
}
```

#### 4）内置类型库 @types
第三方库类型统一托管在 `@types/xxx`，安装即可自动识别：
```bash
npm i @types/lodash -D
```

### 3. tsconfig 关联声明文件
通过 `typeRoots`、`types` 配置读取 `.d.ts`，项目根目录 `global.d.ts` 默认全局生效。

1. **静态类型**：TS 底层核心，所有语法的基础；

3. **函数**：代码复用单元，完善类型校验；
4. **泛型**：实现类型复用，解决重复类型定义；
5. **高级类型**：复杂类型运算，灵活组合类型；
6. **模块/命名空间**：代码分层隔离；
7. **装饰器**：面向切面编程，扩展类能力；
8. **声明文件**：对接 JS 生态，补齐第三方类型。
