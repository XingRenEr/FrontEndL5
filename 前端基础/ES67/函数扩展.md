# ES6 / ES7 函数扩展：尾调用优化 + 函数参数尾逗号
## 42. 尾调用优化（ES6）
### 1. 什么是尾调用
**尾调用**：函数最后一步操作是返回另一个函数的调用，调用位置处于函数尾部。
```js
// 标准尾调用
function fn(x) {
  return bar(x); // 最后一句直接返回函数调用
}

// 非尾调用（调用后还有运算）
function fn(x) {
  return bar(x) + 1; // 调用bar后还要做加法，不是尾调用
}

// 非尾调用（保存中间值）
function fn(x) {
  const res = bar(x);
  return res;
}
```

### 2. 尾递归
尾调用自身，称为**尾递归**，最典型场景：递归计算阶乘、斐波那契。
普通递归会不断堆积调用栈，栈深度过大会栈溢出；尾递归可触发引擎**尾调用优化**。
```js
// 普通递归：栈会叠加
function factorial(n) {
  if (n === 1) return 1;
  return n * factorial(n - 1);
}

// 尾递归版本，可优化
function factorialTail(n, total = 1) {
  if (n === 1) return total;
  return factorialTail(n - 1, n * total); // 尾部调用自身
}
```

### 3. 尾调用优化原理
JS引擎执行函数时会保留**调用帧（栈帧）**，保存内部变量、调用位置；
普通调用：外层函数栈帧会一直保留，直到内层函数执行完运算；
尾调用优化：外层函数执行到末尾，无剩余操作，引擎**销毁外层栈帧**，只保留内层函数栈帧，调用栈永远只有一层，杜绝栈溢出。

### 4. 限制与现状
1. ES6规范规定必须实现该优化，但**绝大多数浏览器、Node.js默认不开启**（V8引擎关闭尾递归优化）；
2. 严格模式 `use strict` 下才会生效，普通模式不触发；
3. 存在闭包、外层变量引用时，无法优化。

## 43. 函数参数的尾逗号（ES2017 / ES7）
### 1. 语法说明
函数定义、函数调用的参数列表最后一个参数后允许添加逗号，不报错。
同时对象、数组字面量也支持尾逗号，ES7统一规范函数参数场景。
```js
// 函数定义末尾加逗号
function sum(a, b, c,) {
  return a + b + c;
}

// 函数调用末尾加逗号
sum(1, 2, 3,);
```

### 2. 解决的实际问题
多人协作、多行格式化参数时，新增参数只需要新增一行，不用修改上一行逗号，减少git变更行数：
```js
// 格式化多行参数，新增参数仅追加一行，不改动原有代码
function login(
  username,
  password,
  captcha, // 末尾逗号保留
) {}

login(
  "admin",
  "123456",
  "8888", // 末尾逗号
);
```

### 3. 对比ES5差异
ES5中函数参数末尾加逗号会直接报语法错误，ES7完全兼容该写法，向下无冲突。

### 4. 配套场景
尾逗号统一支持三类结构：
1. 数组 `[1,2,3,]`
2. 对象 `{a:1, b:2,}`
3. ES7新增：函数形参、实参列表
