# ES6 / ES7 字符串扩展完整详解
## 一、字符串遍历器接口（ES6）
### 1. 核心作用
ES6 为字符串部署了 **Iterator 遍历器接口**，意味着字符串可以使用 `for...of` 循环遍历，完美支持**四字节 Unicode 字符（emoji、生僻汉字）**，解决传统 `for` 循环无法识别 surrogate 代理对的缺陷。

### 2. 传统 for 循环缺陷
```javascript
const str = '😀a';
// 普通for按字节遍历，emoji占2个码元，会拆分乱码
for (let i = 0; i < str.length; i++) {
  console.log(str[i]); // 输出两个空乱码 + a
}
```

### 3. for...of 遍历（基于遍历器）
```javascript
const str = '😀a';
// 自动识别完整Unicode字符，逐个输出字符本体
for (const char of str) {
  console.log(char); 
  // 😀
  // a
}
```

### 4. 其他依赖遍历器的用法
只要部署 Iterator 接口的对象，都能使用：
- 扩展运算符 `[...str]`
- `Array.from(str)`
- `Promise.all([...str].map(...))`

示例：
```javascript
const s = '你好😎';
console.log([...s]); // ['你','好','😎']
console.log(Array.from(s)); // ['你','好','😎']
```

### 5. 底层原理（Symbol.iterator）
字符串原型自带 `Symbol.iterator` 方法，调用生成遍历器：
```javascript
const s = 'ab';
const iterator = s[Symbol.iterator]();
console.log(iterator.next()); // {value:'a', done:false}
console.log(iterator.next()); // {value:'b', done:false}
console.log(iterator.next()); // {value:undefined, done:true}
```

## 二、模板编译（模板字符串实例进阶，ES6）
### 1. 基础模板字符串回顾
`` `字符串${变量}` `` 是基础模板，仅支持简单插值；**模板编译**是把模板字符串转为可复用渲染函数，类似简易前端模板引擎（Vue/React模板底层思路）。

### 2. 什么是模板编译
流程：
1. 解析模板字符串，拆分**静态文本**与 `${表达式}` 插值；
2. 拼接成一段 JS 代码字符串；
3. 通过 `new Function` 动态生成渲染函数；
4. 调用函数传入数据，返回完整渲染后的字符串。

### 3. 简易模板编译实现
```javascript
// 模板编译函数
function compileTemplate(tpl) {
  // 1. 把模板替换成拼接代码：`xxx${name}yyy` → 'xxx' + data.name + 'yyy'
  let code = tpl.replace(/\$\{(.+?)\}/g, "' + data.$1 + '");
  // 2. 首尾拼接引号，形成完整字符串表达式
  code = `return '${code}';`;
  // 3. 生成渲染函数，接收data数据
  return new Function('data', code);
}

// 使用
const template = `
  <div>
    <h1>${title}</h1>
    <p>${content}</p>
  </div>
`;
// 编译模板得到渲染函数
const render = compileTemplate(template);
// 传入数据渲染
const html = render({ title: 'ES6教程', content: '模板编译详解' });
console.log(html);
```

### 4. 实际业务价值
- 模板只编译一次，多次渲染复用，性能更高；
- 实现页面模板与数据分离，不用频繁拼接字符串；
- 框架模板（Vue SFC、Lit）底层均是**模板编译**思路，将模板转执行代码。

### 5. 缺陷与优化方向
原生简易编译无法处理：换行转义、单双引号冲突、复杂表达式、循环；工程中会搭配正则转义、AST 解析优化。

## 三、标签模板（Tagged Template，ES6 核心重点）
### 1. 定义
**标签模板 = 函数 + 模板字符串**
语法：
```js
标签函数`模板内容${变量1}${变量2}`
```
等价于自动调用标签函数，参数由引擎自动拆分传递，不需要括号 `()`。

### 2. 标签函数接收的参数规则（核心）
1. **第一个参数：静态字符串数组**
   所有插值分割后的固定文本，数组自带 `raw` 属性，存储原始未转义字符串；
2. **后续参数：依次对应每个 `${}` 插值表达式的值**

示例拆解：
```javascript
tag`a${x}b${y}c`
// 引擎自动转译为：
tag(['a','b','c'], x, y)
```

### 3. 基础示例演示
```javascript
// 定义标签函数
function tag(strs, ...values) {
  console.log('静态文本数组：', strs);
  console.log('原始文本raw：', strs.raw);
  console.log('插值变量：', values);
  // 手动拼接还原完整字符串
  let res = '';
  strs.forEach((s, i) => {
    res += s + (values[i] ?? '');
  });
  return res;
}

const name = '小明';
const age = 18;
const result = tag`姓名：${name}，年龄：${age}`;
/* 输出打印：
静态文本数组：[ '姓名：', '，年龄：', '' ]
原始文本raw：[ '姓名：', '，年龄：', '' ]
插值变量：[ '小明', 18 ]
*/
console.log(result); // 姓名：小明，年龄：18
```

### 4. strs.raw 原始字符串（关键特性）
普通字符串会转义 `\n \t \\`，`.raw` 保留转义符原始形态：
```javascript
function rawTag(str) {
  console.log(str[0]); // 换行被转义，直接换行
  console.log(str.raw[0]); // 保留原始 \n
}
rawTag`hello\nworld`
// hello
// world
// hello\nworld
```
**常用场景**：写正则、文件路径、多行带转义符文本。

### 5. 标签模板经典实战场景
#### 场景1：HTML 防 XSS 过滤（最常用）
自动转义 `< > & " '`，防止注入攻击：
```javascript
function safeHtml(strs, ...vals) {
  const escape = s => s.replace(/[&<>"']/g, c => ({
    '&':'&amp;','<':'&lt;','>':'&gt;','"':'&quot;',"'":'&#39;'
  }[c]));
  return strs.reduce((html, s, i) => {
    return html + s + (vals[i] ? escape(vals[i]) : '');
  }, '');
}

const userInput = '<script>alert("攻击")</script>';
const html = safeHtml`<div>${userInput}</div>`;
console.log(html); 
// <div>&lt;script&gt;alert(&quot;攻击&quot;)&lt;/script&gt;</div>
```

#### 场景2：多语言国际化 i18n
```javascript
function i18n(strs, ...vals) {
  const map = { '你好': 'Hello', '年龄': 'Age' };
  let text = strs.join('{v}');
  Object.keys(map).forEach(k => text = text.replace(k, map[k]));
  for (const v of vals) text = text.replace('{v}', v);
  return text;
}
const a = 20;
console.log(i18n`你好，年龄：${a}`); // Hello，Age：20
```

#### 场景3：Styled-components CSS 样式（React生态）
```jsx
// 第三方库底层就是标签模板
const Box = styled`
  width: ${props => props.w}px;
  color: red;
`
```

#### 场景4：正则简化、SQL 防注入
SQL 标签模板可自动给插值加引号、过滤非法字符，避免 SQL 注入漏洞。

### 6. ES7 补充说明
以上三个特性**全部属于 ES6（ES2015）**，ES7（ES2016）字符串仅新增一个 API：
- `String.prototype.includes()` / startsWith / endsWith 是 ES6
- ES7 字符串无新增语法扩展，仅数组新增 `Array.prototype.includes`

## 四、三者总结对比
| 特性                     | 版本                      | 核心用途                                          |
| ------------------------ | ------------------------- | ------------------------------------------------- |
| 字符串遍历器接口         | ES6                       | for...of 完整遍历Unicode字符，支持扩展运算符      |
| 模板编译                 | ES6（模板字符串衍生用法） | 将模板预编译为渲染函数，实现模板引擎              |
| 标签模板 Tagged Template | ES6                       | 拦截模板插值，统一处理文本（XSS、i18n、CSS、SQL） |

### 核心考点速记
1. 遍历器解决 emoji/生僻字遍历乱码，依赖 `Symbol.iterator`；
2. 模板编译本质是动态生成渲染函数，分离模板与数据；
3. 标签模板第一个参数为静态字符串数组，带 `.raw`，后面依次是插值，语法不用括号。
