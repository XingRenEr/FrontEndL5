# ES6 / ES7 正则扩展完整详解
## 前置说明
- ES6（ES2015）新增：`RegExp`构造函数传参优化、字符串正则4个方法统一调度、`u`修饰符、`y`修饰符
- ES2018（常统称ES7+）新增：`s`修饰符（dotAll模式）
下面按你列出的5个点逐一拆解，配对比代码、使用场景、坑点。

# 4. RegExp 构造函数扩展
## 旧版 ES5 限制
ES5 有两种创建正则写法，但**两种参数不能混用**：
1. 传字符串：`new RegExp('正则', '修饰符')`
2. 传正则实例：`new RegExp(/abc/g)`
ES5 中，第二个参数修饰符**完全无效**，无法覆盖原有正则的修饰符。
```js
// ES5
const reg1 = new RegExp(/abc/g, 'i');
console.log(reg1.flags); // "g"，第二个参数i被忽略
```

## ES6 修复优化
ES6 允许第二个参数修饰符**覆盖**原有正则实例的修饰符，优先级更高。
```js
// ES6
const reg2 = new RegExp(/abc/g, 'i');
console.log(reg2.flags); // "i"，成功覆盖
```

## 两种创建方式完整示例
### 方式1：两个字符串参数（兼容ES5/6）
```js
const r1 = new RegExp('hello\\d', 'gi');
// 等价字面量 /hello\d/gi
```
> 注意字符串内`\`需要转义，写`\\d`。

### 方式2：正则实例 + 修饰符（ES6独有增强）
```js
const origin = /test/m;
const r2 = new RegExp(origin, 'ig');
console.log(r2.source); // test
console.log(r2.flags);  // ig 覆盖原来的m
```

## 补充属性
ES6 给正则实例新增只读属性：
- `reg.flags`：返回所有修饰符字符串（g/i/m/u/y/s）
- `reg.source`：返回正则本体文本

# 5. 字符串的正则方法统一调度
## ES5 原有4个字符串方法，都支持正则参数
`match()`、`replace()`、`search()`、`split()`
ES6 统一重构底层逻辑：**全部内部调用 `RegExp` 原型上的对应方法**，标准化行为。

| 字符串方法               | 内部调用正则原型方法             | 作用                     |
| ------------------------ | -------------------------------- | ------------------------ |
| str.match(reg)           | RegExp.prototype[Symbol.match]   | 匹配字符串，返回匹配数组 |
| str.replace(reg, fn/str) | RegExp.prototype[Symbol.replace] | 替换匹配内容             |
| str.search(reg)          | RegExp.prototype[Symbol.search]  | 返回首个匹配下标，无则-1 |
| str.split(reg)           | RegExp.prototype[Symbol.split]   | 按匹配内容分割数组       |

## 关键变化：传字符串会自动转为正则
如果传入普通字符串，引擎自动 `new RegExp` 转换：
```js
'123abc'.match('\\d+'); 
// 等价 '123abc'.match(/\d+/)
```

## 示例对比
```js
const str = 'a1b2c3';
// match
console.log(str.match(/\d/g)); // ["1","2","3"]
// search
console.log(str.search(/b/));  // 2
// replace
console.log(str.replace(/\d/, '*')); // a*b2c3
// split
console.log(str.split(/\d/)); // ["a","b","c",""]
```

## 核心意义
统一底层接口后，自定义正则类重写`Symbol.match`等即可拦截字符串方法，实现自定义匹配逻辑。

# 6. u 修饰符（unicode 修饰符 ES6）
## 解决的问题
JS 字符编码基于 UTF-16，生僻汉字、emoji、四字节Unicode字符会被拆成**两个码元**。
不加`u`时，正则会把4字节字符当成两个独立字符匹配，导致匹配失效。

## 作用
`u` 开启 Unicode 模式，正则把 `\uXXXX`、四字节字符视为完整单个字符，支持：
1. 正确匹配4字节Unicode字符
2. 支持 `\u{xxxx}` 大括号Unicode写法
3. 量词、边界匹配识别完整字符

## 示例1：不加u匹配失败
```js
// 😀 是四字节UTF16字符
console.log(/^.$/.test('😀')); // false
// 正则把emoji拆成2个码元，^.$ 只能匹配单个码元
```

## 示例2：加u修饰符正常匹配
```js
console.log(/^.$/u.test('😀')); // true
```

## 示例3：大括号Unicode语法必须带u
```js
// 匹配汉字“𠮷”，Unicode码 U+20BB7
/\u{20BB7}/u.test('𠮷'); // true
/\u{20BB7}/.test('𠮷'); // 报错，无u不支持大括号
```

## 配套正则特性（依赖u）
- `\p{}` Unicode属性类（ES2018，同样需要u）
- 字符范围正确识别四字节字符
- `.` 不会截断四字节字符

# 7. y 修饰符（sticky 粘连修饰符 ES6）
## 核心概念：粘连匹配
`g` 全局修饰符：**任意位置**继续向后匹配
`y` 粘连修饰符：**必须从 lastIndex 下标处紧贴匹配**，不能跳过字符

正则实例有 `lastIndex` 属性，记录下一次匹配起始位置。

## 对比 g 和 y
```js
const str = 'aaa_aa';
// 全局g
const regG = /aa/g;
console.log(regG.exec(str)); // [0,2] 下标0匹配
console.log(regG.exec(str)); // [4,6] 跳过下划线直接匹配后面aa

// 粘连y
const regY = /aa/y;
console.log(regY.exec(str)); // [0,2] lastIndex=2
console.log(regY.exec(str)); // null，下标2是a，下一位是_，无法紧贴匹配
```

## y 修饰符关键规则
1. 每次匹配严格从 `lastIndex` 位置开始，中间不能有其他字符
2. 匹配成功：`lastIndex` 更新为匹配结束下标
3. 匹配失败：`lastIndex` 重置为0
4. 单独使用无意义，一般配合循环解析分段字符串

## 实战场景：分段解析模板、语法分词（Lexer）
```js
function tokenize(code) {
  const tokenReg = /\w+|,|;/y;
  const tokens = [];
  let res;
  while ((res = tokenReg.exec(code))) {
    tokens.push(res[0]);
  }
  return tokens;
}
console.log(tokenize('name,age;')); // ["name", ",", "age", ";"]
```
适合简单词法分析，防止跨段非法匹配。

## 补充属性
`reg.sticky` 布尔值，判断是否带y修饰符：
```js
/abc/y.sticky; // true
/abc/g.sticky; // false
```

# 8. s 修饰符：dotAll 模式（ES2018，ES7系列）
## 传统正则痛点
元字符 `.` 匹配**任意单个字符，但不匹配换行符 `\n \r`**
多行文本、模板字符串无法用 `.+` 一次性匹配整块内容。

## s 修饰符作用
开启 `dotAll` 模式，`.` 可以匹配**换行符**。

## 对比演示
```js
const multi = `hello
world`;
// 无s：. 不能跨换行
console.log(/hello.+world/.test(multi)); // false
// 加s修饰符
console.log(/hello.+world/s.test(multi)); // true
```

## 和 m 修饰符区分（极易混淆考点）
- `m`（multiline 多行）：只改变 `^` 和 `$`，匹配每行开头结尾，不影响 `.`
- `s`（dotAll）：只改变 `.`，允许匹配换行，不影响 `^ $`
两者可同时使用 `/^.+$/sm`

```js
const str = 'a\nb';
// m修饰符：^匹配每行开头
console.log(str.match(/^b/m)); // ["b"]
// s修饰符：.包含换行
console.log(str.match(/a.+b/s)); // ["a\nb"]
```

## 配套只读属性
`reg.dotAll` 判断是否开启s修饰符：
```js
/./s.dotAll; // true
/./g.dotAll; // false
```

# 高频面试考点速记
1. ES5 RegExp第二个修饰符参数无效，ES6可覆盖；
2. u解决emoji/生僻字匹配错乱，y严格粘连不跳过字符，s让点匹配换行；
3. m改^$，s改`.`，二者互不冲突；
4. y修饰符依靠lastIndex，失败自动归零；
5. 字符串四个正则方法底层统一调用正则原型Symbol内置方法。
