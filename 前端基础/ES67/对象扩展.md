# ES6 对象扩展：方法name属性 + super关键字 完整详解
## 一、对象方法的 name 属性（ES6）
### 1. 基础概念
函数都有 `name` 属性，返回函数名；ES6 为**对象简写方法**、**get/set 存取器**、**bind 绑定函数**、**Symbol 方法**统一规范了 `name` 取值规则。
对象字面量简写方法，本质是函数，可直接读取 `.name`。

### 2. 各类场景示例
#### （1）普通简写方法
```js
const obj = {
  sayHello() {},
};
console.log(obj.sayHello.name); // "sayHello"
```

#### （2）普通函数赋值（区别简写）
```js
const obj = {
  fn: function test() {},
  anon: function () {},
};
console.log(obj.fn.name); // "test"
console.log(obj.anon.name); // "" 空字符串
```

#### （3）get / set 存取器方法
getter、setter 的 `name` 会带前缀 `get`/`set`，中间空格分隔：
```js
const person = {
  get name() {},
  set name(val) {},
};
const descriptor = Object.getOwnPropertyDescriptor(person, "name");
console.log(descriptor.get.name); // "get name"
console.log(descriptor.set.name); // "set name"
```

#### （4）Generator 生成器方法
```js
const obj = {
  *gen() {},
};
console.log(obj.gen.name); // "gen"
```

#### （5）Symbol 作为方法名
函数名返回 Symbol 的描述文本：
```js
const sym = Symbol("demo");
const obj = {
  [sym]() {},
};
console.log(obj[sym].name); // "[demo]"
```

#### （6）bind 绑定后的方法
name 追加 `bound ` 前缀：
```js
function func() {}
const obj = { fn: func.bind(null) };
console.log(obj.fn.name); // "bound func"
```

### 3. 实用场景
- 日志打印、错误堆栈区分函数；
- 反射、框架内部识别对象方法；
- 装饰器中获取原始方法名称。

### 4. 补充说明
- 匿名函数赋值给对象属性，`name` 为空字符串；
- ES5 无统一规范，ES6 标准化所有对象方法的 name 规则。

## 二、super 关键字（ES6，仅在对象简写方法 / class 中可用）
### 1. 核心作用
`super` 代表**原型对象**，分两种用法：
1. `super.xxx`：读取原型上的属性/方法；
2. `super(...)`：仅 class 构造函数调用父类构造器（对象字面量不可用）。

⚠️ 重要限制：**super 只能写在 ES6 对象简写方法中，普通 function 函数内不能使用 super**。

### 2. 语法限制对比
```js
const parent = {
  fn() {
    console.log("父方法");
  },
};
const child = {
  // 简写方法 ✅ 支持super
  fn() {
    super.fn();
  },
  // 普通函数 ❌ 报错：super unexpected
  bad: function () {
    super.fn();
  },
};
// 绑定原型
Object.setPrototypeOf(child, parent);
child.fn(); // 父方法
child.bad(); // SyntaxError
```

### 3. super 查找原理
`super.xxx` 等价于：
```js
Object.getPrototypeOf(this).xxx;
```
执行时自动以当前 `this` 的原型链为查找目标，不用手动获取原型。

### 4. 完整示例：对象字面量继承
```js
// 原型对象
const animal = {
  type: "动物",
  speak() {
    console.log(`我是${this.type}`);
  },
};

// 子对象，原型指向 animal
const dog = {
  type: "小狗",
  speak() {
    // 调用原型 speak 方法
    super.speak();
    console.log("汪汪汪");
  },
};
Object.setPrototypeOf(dog, animal);

dog.speak();
// 输出：
// 我是小狗
// 汪汪汪
```
解析：`super.speak()` 调用原型 `animal.speak`，但函数内部 `this` 仍然是 `dog`，所以 `this.type` 取到“小狗”。

### 5. super 赋值：super.xxx = yyy
给 super 属性赋值，**修改的是当前实例自身属性，不是原型**：
```js
const parent = { name: "父" };
const child = {
  setName() {
    super.name = "子";
  },
};
Object.setPrototypeOf(child, parent);
child.setName();
console.log(child.name); // "子"（自身新增属性）
console.log(parent.name); // "父"（原型不变）
```
规则：
- `super.xxx = val`：赋值给 `this`（当前对象）；
- `const val = super.xxx`：读取原型上的 xxx。

### 6. class 中 super 补充（联动知识点）
class 内 super 两种用法：
1. 构造器 `super()`：调用父类构造函数，必须在子类 constructor 第一行；
2. 方法 `super.方法名()`：调用父类原型方法。
```js
class Animal {
  constructor(name) {
    this.name = name;
  }
  say() {
    console.log(this.name);
  }
}
class Dog extends Animal {
  constructor() {
    super("旺财"); // 调用父构造器
  }
  say() {
    super.say(); // 调用父类方法
    console.log("汪汪");
  }
}
new Dog().say();
```

### 7. super 常见易错点
1. 不能在普通 function、箭头函数中使用 super；
2. 箭头函数不绑定 this，内部使用 super 会报错；
```js
const obj = {
  fn: () => {
    super.test(); // 报错
  },
};
```
3. super 依赖当前对象的原型，修改原型后 super 指向同步变化；
4. super 不能单独使用，必须搭配 `.` 访问属性/方法。

### 8. 应用场景
1. 对象字面量实现简单原型继承，重写方法时复用父逻辑；
2. class 类继承，扩展父类功能；
3. 多层原型链调用上层同名方法，避免手动 `Object.getPrototypeOf` 冗长写法。

# 总结速记
1. **方法 name 属性**
   - 对象简写方法自带 name，返回方法名；
   - get/set 名称带 `get/set` 前缀，Symbol 方法名包裹 `[]`；
   - 绑定函数 name 带 `bound` 前缀。

2. **super 关键字**
   - 仅 ES6 简写方法 / class 可用；
   - `super.xxx` 读原型属性，赋值则修改自身；
   - `super()` 仅 class 构造函数可用；
   - 内部 this 指向调用者，而非原型对象。
