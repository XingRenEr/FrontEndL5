# ES6 / ES7 数值扩展完整详解
全部特性均为 **ES2015（ES6）** 新增，无ES7专属数值扩展，下面按6个模块逐一对比ES5旧API、原理、示例、面试坑点。

## 9. Number.isFinite() 判断是否为有限数值
### 1. ES5 旧方案缺陷：全局 isFinite()
全局 `isFinite()` 会**自动参数转数字**，存在大量误判：
```js
isFinite('123'); // true 字符串被转数字
isFinite(true);  // true true→1
isFinite(null);  // true null→0
isFinite(Infinity); // false
isFinite(NaN);     // false
```

### 2. ES6 Number.isFinite() 规则
**不做类型强制转换**，只有严格是 `Number` 类型且**不是 Infinity / -Infinity / NaN** 才返回 true。
判断逻辑：
1. 参数类型必须是数字 `typeof x === 'number'`
2. 不能是正负无穷、不能是NaN

```js
console.log(Number.isFinite(15));      // true
console.log(Number.isFinite(0.8));     // true
console.log(Number.isFinite(NaN));     // false
console.log(Number.isFinite(Infinity));// false
console.log(Number.isFinite(-Infinity));//false

// 非数字一律false，不会隐式转换
console.log(Number.isFinite('15'));    // false
console.log(Number.isFinite(true));    // false
console.log(Number.isFinite(null));    // false
console.log(Number.isFinite({}));      // false
```

### 核心区别总结
- 全局 `isFinite`：先调用 `Number(x)` 转数字再判断
- `Number.isFinite`：先判断类型是否为number，再判断有限性

## 10. Number.isNaN() 判断是否严格为NaN
### ES5 全局 isNaN() 巨大缺陷
全局方法会强制转数字，任何无法转为有效数字的值都会判定为NaN，极易出错：
```js
isNaN('abc'); // true 字符串转NaN，误判
isNaN('123'); // false
isNaN(true);  // false
```

### ES6 Number.isNaN() 严格判断
只有**值严格等于 NaN** 才返回 true，其余全部 false。
判断逻辑：`typeof x === 'number' && x !== x`
```js
console.log(Number.isNaN(NaN));        // true
console.log(Number.isNaN(0 / 0));      // true

// 非数字全部false
console.log(Number.isNaN('NaN'));      // false
console.log(Number.isNaN('abc'));      // false
console.log(Number.isNaN(123));        // false
console.log(Number.isNaN(true));       // false
```

### 面试重点：NaN唯一特性
JS 中只有 `NaN` 自己不等于自己：`NaN !== NaN`，底层就是依靠这个特性实现判断。

## 11. Number.parseInt() / Number.parseFloat()
### ES5 写法：全局函数
```js
parseInt('12.3abc');  // 12
parseFloat('12.3abc');// 12.3
```
### ES6 改动：挂载到Number静态方法
ES6 将全局的 `parseInt、parseFloat` 移植到 `Number` 对象上，行为**完全一致**，无任何逻辑修改。
```js
Number.parseInt('12.3abc');   // 12
Number.parseFloat('12.3abc'); // 12.3

// 全等验证，底层是同一个函数
console.log(Number.parseInt === parseInt); // true
console.log(Number.parseFloat === parseFloat); // true
```

### 设计目的
语言模块化，把数值相关API统一收敛到 `Number` 对象，减少全局顶层污染，规范API归类。

## 12. Number.isInteger() 判断是否为整数
### 判断规则
1. 类型必须是 `number`
2. 数字在数学意义上是整数（小数点后无有效值）
3. 支持正负整数、0；浮点数小数为0也判定整数

```js
console.log(Number.isInteger(25));     // true
console.log(Number.isInteger(25.0));   // true
console.log(Number.isInteger(25.1));    // false

// 特殊值
console.log(Number.isInteger(0));      // true
console.log(Number.isInteger(-0));     // true
console.log(Number.isInteger(NaN));    // false
console.log(Number.isInteger(Infinity));//false

// 非数字直接false，不隐式转换
console.log(Number.isInteger('25'));   // false
console.log(Number.isInteger(true));   // false
```

### 边界坑点：超大浮点数精度丢失
JS 安全整数范围：`-2^53 ~ 2^53`，超出后浮点精度失效，会出现误判：
```js
// 超出安全整数，存储精度丢失，被判定为整数
console.log(Number.isInteger(9007199254740993)); // true
```
配套：`Number.MAX_SAFE_INTEGER` / `Number.MIN_SAFE_INTEGER`、`Number.isSafeInteger()`

## 13. Number.EPSILON 极小常量，解决浮点数精度误差
### 背景问题
JS 二进制浮点数运算存在精度丢失，直接相等判断会出错：
```js
console.log(0.1 + 0.2 === 0.3); // false
```

### EPSILON 定义
`Number.EPSILON` 表示 1 与大于 1 的最小浮点数之间的差值，值约 `2.220446049250313e-16`。
作用：设定**误差阈值**，两个浮点数差值小于该值，则认为相等。

### 精度对比通用函数封装
```js
function equal(a, b) {
  return Math.abs(a - b) < Number.EPSILON;
}

console.log(equal(0.1 + 0.2, 0.3)); // true
console.log(equal(0.1 + 0.2, 0.4)); // false
```

### 使用场景
金融计算、图形坐标、科学计算等需要对比浮点数相等的场景。

## 14. Math 对象的扩展（ES6新增17个静态方法）
所有方法均为静态方法，只能通过 `Math.xxx()` 调用，下面分类整理常用核心方法。

### 一、取整系列（解决 parseInt 负数缺陷）
1. **Math.trunc()**：直接去除小数部分，不四舍五入
```js
Math.trunc(4.9);  // 4
Math.trunc(-4.9); // -4
Math.trunc(-0.1); // -0
// 非数字先转Number
Math.trunc('12.3');//12
Math.trunc(NaN);   // NaN
```
对比：
- `Math.floor()`：向下取整，负数会变小 `floor(-4.9) = -5`
- `Math.ceil()`：向上取整
- `Math.round()`：四舍五入

2. **Math.sign()**：判断数字符号
返回值：
- 1：正数
- -1：负数
- 0：+0
- -0：-0
- NaN：非数字
```js
Math.sign(5);     // 1
Math.sign(-5);    // -1
Math.sign(0);     // 0
Math.sign(-0);    // -0
Math.sign('abc'); // NaN
```

### 二、立方根、对数、指数
3. Math.cbrt(x) 立方根
```js
Math.cbrt(8);  // 2
Math.cbrt(-8); // -2
```
4. Math.hypot(a,b,c...) 返回所有参数平方和的平方根（勾股定理）
```js
Math.hypot(3,4); // 5
```
5. Math.expm1(x) = eˣ - 1
6. Math.log1p(x) = ln(1+x)
7. Math.log10(x) 以10为底对数
8. Math.log2(x) 以2为底对数

### 三、二进制/32位运算相关
9. Math.clz32(x)：32位无符号整数前置0的个数
10. Math.imul(a,b)：32位整数乘法，修复大数乘法溢出精度丢失

### 四、双曲函数（极少业务使用）
Math.sinh、Math.cosh、Math.tanh、Math.asinh、Math.acosh、Math.atanh

### 五、ES6 新增幂运算符补充（配套数值）
`**` 幂运算，等价 `Math.pow`
```js
2 ** 3 === Math.pow(2,3) // 8
```

| -------------------------- | ------------ | ------------------------------ |

| Number.isNaN               | 判断严格NaN  | 只有NaN返回true                |

| Math.hypot                 | 平方和开根号 | 多参数勾股计算                 |

# 高频面试考点
1. `Number.isFinite / isNaN` 和全局同名函数最大区别：**不会自动隐式转换参数类型**；
2. 判断浮点数相等不能直接 `===`，要用 `Number.EPSILON` 做误差比较；
3. `Math.trunc` 区别 floor：负数不会向下取值；
4. `Number.isInteger(25.0)` 结果为 true；
5. ES6 将 parseInt 挂载 Number，只是归类优化，逻辑无变化。
