# JS 对象四大核心知识点：访问器、构造器、方法、原型
## 23. 对象 — 访问器属性（get / set）
普通属性是**数据属性**，`get/set` 是**访问器属性**，拦截读写操作。
### 1. 字面量写法
```js
const user = {
  _age: 18, // 约定下划线私有，外部不要直接修改
  // 读取时触发
  get age() {
    return this._age
  },
  // 赋值时触发
  set age(val) {
    if (typeof val !== 'number' || val < 0) {
      throw new Error('年龄必须是正数')
    }
    this._age = val
  }
}

console.log(user.age) // 自动调用get，输出18
user.age = 20 // 自动调用set
user.age = -5 // 报错
```

### 2. Object.defineProperty 定义访问器
```js
const obj = {}
let _name = ''
Object.defineProperty(obj, 'name', {
  get() { return _name },
  set(v) { _name = v }
})
obj.name = '张三'
console.log(obj.name)
```

### 特点
- 访问器**不能和 value、writable 共存**；
- 只读只写：只写 get → 只读；只写 set → 无法读取；
- 常用于数据校验、缓存、联动修改其他属性。

## 24. 对象 — 构造器（构造函数 Constructor）
### 1. 基础构造函数
大写命名，配合 `new` 创建实例对象，`this` 指向新实例。
```js
function Person(name) {
  // 实例自有属性
  this.name = name
  this.say = function() {
    console.log(this.name)
  }
}

// new 执行四步
// 1. 创建空新对象
// 2. 绑定 this 到新对象
// 3. 执行构造函数代码
// 4. 默认返回新对象
const p1 = new Person('小明')
console.log(p1.name) // 小明
```

### 2. class 类语法糖（本质仍是构造函数）
```js
class Student {
  constructor(name) { // 构造器，实例化必执行
    this.name = name
  }
}
const s = new Student('小红')
```

### 3. constructor 属性
每个实例自带 `constructor`，指向它的构造函数：
```js
console.log(p1.constructor === Person) // true
```

### 4. 无 new 调用构造函数坑
直接调用时 `this` 指向 window/undefined（严格模式），不会生成对象。

## 25. 对象 — 对象方法
方法就是**值为函数**的对象属性，分三种写法：
### 1. 对象字面量简写（推荐）
```js
const cat = {
  name: '橘猫',
  // 简写方法
  meow() {
    console.log(this.name + ' 喵喵')
  },
  // 异步方法
  async run() {}
}
cat.meow()
```

### 2. 完整键值写法
```js
const dog = {
  bark: function() {
    console.log('汪汪')
  }
}
```

### 3. 箭头函数不能做对象方法
箭头函数无自己的 `this`，会继承外层作用域：
```js
const bird = {
  name: '麻雀',
  fly: () => {
    console.log(this.name) // undefined，this 不是bird
  }
}
```

### 实例方法 vs 静态方法
- 实例方法：定义在构造函数原型 / 对象字面量，实例调用；
- 静态方法：`Person.fn()`，挂载构造函数本身，不能用实例调用。

```js
class Person {
  // 实例方法
  say() {}
  // 静态方法
  static create() {}
}
Person.create()
new Person().say()
```

## 26. 对象 — 原型 prototype / __proto__
### 1. 两大核心概念
1. `构造函数.prototype`：**显式原型**，存放共享方法；
2. `实例.__proto__`：**隐式原型**，指向构造函数的 prototype；
3. 原型链：对象查找属性/方法，自身没有就顺着 `__proto__` 往上找。

### 基础示例
```js
function Person(name) {
  this.name = name
}

// 挂载原型共享方法（所有实例共用一份，节省内存）
Person.prototype.say = function() {
  console.log(this.name)
}

const p1 = new Person('小李')
console.log(p1.__proto__ === Person.prototype) // true
console.log(Person.prototype.constructor === Person) // true

p1.say() // 自身无say，去原型找到并执行
```

### 2. 原型链查找规则
```js
// p1 → Person.prototype → Object.prototype → null
console.log(p1.toString()) // toString 在 Object 原型上
```

### 3. class 下的原型（语法糖底层一样）
```js
class Person {
  say() {} // 自动挂载到 Person.prototype
}
```

### 4. 常用原型API
- `obj.hasOwnProperty('key')`：判断属性是否为**实例自有**，不查原型；
- `Object.getPrototypeOf(obj)`：标准获取隐式原型（替代 __proto__）；
- `Object.setPrototypeOf(obj, proto)`：修改对象原型。

```js
console.log(p1.hasOwnProperty('name')) // true
console.log(p1.hasOwnProperty('say')) // false（在原型）
```

## 四者快速区分总结
1. **访问器 get/set**：控制对象属性读写拦截、数据校验；
2. **构造器 constructor**：批量创建同类对象的模板函数，new 生成实例；
3. **对象方法**：对象内的函数，用于描述对象行为；
4. **原型 prototype**：实现实例方法共享、构成原型链、JS 继承底层原理。
