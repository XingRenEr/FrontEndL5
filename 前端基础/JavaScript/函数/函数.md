# JS 函数核心知识点（27~30）
## 27. 函数 & 函数调用
### 1. 函数定义三种方式
```js
// 1. 函数声明
function fn() {}

// 2. 函数表达式
const fn = function() {}

// 3. 箭头函数
const fn = () => {}
```
### 2. 函数调用方式
1. **普通调用**：`fn()`，this 指向 window（严格模式 undefined）
2. **对象方法调用**：`obj.fn()`，this 指向调用对象 obj
3. **构造函数调用**：`new Fn()`，this 指向新实例
4. **间接调用**：`call / apply / bind`

### 形参、实参、arguments
- 实参数量 > 形参：多余参数存在 `arguments`
- 实参 < 形参：未传参数为 `undefined`
- `arguments` 类数组，箭头函数无 arguments

## 28. call()
### 语法
`函数.call(thisArg, 参数1, 参数2, ...)`
### 作用
1. 改变函数内部 `this` 指向
2. **逐个传递参数**
3. 立即执行函数

### 示例
```js
function sum(a, b) {
  console.log(this, a + b)
}
const obj = { name: "张三" }
sum.call(obj, 10, 20) // this=obj，输出30
```

### 经典用途：类数组转数组
```js
let arr = Array.prototype.slice.call(arguments)
```

## 29. apply()
### 语法
`函数.apply(thisArg, [参数数组])`
### 和 call 区别
- call：参数**逐个传**
- apply：参数**数组/类数组统一传入**
### 示例
```js
sum.apply(obj, [10, 20])

// 快速求数组最大值
const arr = [1,5,9]
Math.max.apply(null, arr)
```

### 共同点
1. 都会立刻执行函数
2. 第一个参数都用于绑定 this
3. 不传 thisArg / 传 null/undefined，非严格模式下 this 指向 window

## 30. 闭包 Closure
### 定义
**函数 A 内部嵌套函数 B，B 访问了 A 的局部变量，B 被外部使用，就形成闭包**。
核心：内部函数持有外部函数作用域的引用，外部函数执行完变量不会销毁。

### 形成条件
1. 函数嵌套
2. 内部函数引用外部函数局部变量/参数
3. 内部函数暴露到外部执行

### 基础示例
```js
function outer() {
  let num = 10
  // 内部函数
  function inner() {
    console.log(num) // 引用外层变量
  }
  return inner // 暴露出去
}
const fn = outer()
fn() // 输出10，闭包生效
```

### 闭包作用
1. **私有化变量**：外部无法直接修改，只能通过内部方法操作
```js
function counter() {
  let count = 0
  return {
    add() { count++ },
    get() { return count }
  }
}
const c = counter()
c.add()
console.log(c.get()) // 1
// 无法直接访问 count
```
2. **保存变量状态**，变量常驻内存
3. 模块化、防抖节流、定时器回调

### 闭包缺点
变量长期驻留内存，容易**内存泄漏**，不用时手动释放引用：`fn = null`

### 经典面试题：循环+闭包
```js
for(var i=0; i<3; i++){
  setTimeout(()=>console.log(i), 0)
}
// 输出 3 3 3
// 解决：let 块级作用域 / 立即执行函数形成闭包
```
