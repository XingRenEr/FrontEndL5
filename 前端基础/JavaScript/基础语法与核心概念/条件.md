# JS 条件判断完整语法
## 一、if / else if / else 基础分支
### 语法
```js
if (条件表达式) {
  // 条件为真值执行
} else if (条件2) {
  // 条件1假、条件2真执行
} else {
  // 全都不满足执行
}
```
### 示例
```js
let score = 85;
if (score >= 90) {
  console.log("优秀");
} else if (score >= 60) {
  console.log("及格");
} else {
  console.log("不及格");
}
```
### 关键点
括号内表达式会自动转布尔（6个假值直接走else）：
`false、0、""、null、undefined、NaN`

## 二、三元表达式（简写if else）
语法：`条件 ? 成立返回值 : 不成立返回值`
### 基础
```js
let age = 18;
let tip = age >= 18 ? "成年" : "未成年";
```
### 多层嵌套（不推荐写太长）
```js
let s = 80;
let level = s >=90 ? 'A' : s>=60 ? 'B' : 'C';
```

## 三、switch 多分支匹配
### 基础语法
```js
switch(表达式) {
  case 值1:
    代码;
    break; // 不加break会穿透到下一个case
  case 值2:
    代码;
    break;
  default: // 都不匹配执行
    默认代码;
}
```
### 示例（严格匹配 ===）
```js
let num = 2;
switch(num) {
  case 1:
    console.log("一");
    break;
  case 2:
    console.log("二");
    break;
  default:
    console.log("其他");
}
```
### 特性
1. 匹配是**严格相等 ===**，不会隐式转换
2. `break` 阻断穿透；穿透可合并多个case
```js
// 合并多个条件
let n = 3;
switch(n) {
  case 1:
  case 2:
  case 3:
    console.log("1-3区间");
    break;
}
```

## 四、短路条件简写（&& / || / ??）
### 1. && 满足才执行（替代单行if）
```js
let name = "张三";
name && console.log("有名字");
// 等价于 if(name) { console.log("有名字") }
```
### 2. || 兜底默认值
```js
let username = "";
username = username || "游客";
```
### 3. ?? 空值兜底（只判断null/undefined）
```js
let count = 0;
count = count ?? 10; // 0保留，不会覆盖
```

## 五、可选链 ??. 安全条件判断
防止读取不存在属性报错，配合??做默认值
```js
const user = null;
const nick = user?.info?.nickname ?? "匿名";
```

## 六、if 常见坑
1. `=` 赋值和 `== ===` 混淆
```js
// 错误，永远true
if(x = 10){}
// 正确判断相等
if(x === 10){}
```
2. 空数组/空对象是真值
```js
if([]) console.log("会执行"); // []为true
if({}) console.log("会执行"); // {}为true
```
3. NaN判断不能直接 ==
```js
let a = NaN;
if(a === NaN) {} // false
if(Number.isNaN(a)) {} // 正确
```
4. 分号陷阱
```js
if(1>0);{
  console.log("永远执行");
}
// 分号代表if语句结束，大括号代码不受条件控制
```

## 七、四种条件场景选用建议
1. 2种分支：三元表达式
2. 3种以内多判断：if else if
3. 固定等值匹配、大量分支：switch
4. 简单存在判断、默认值：短路 && || ??
