# JS 正则表达式完整教程
## 一、两种创建方式
### 1. 字面量（推荐，固定正则）
`/匹配规则/修饰符`
```js
const reg = /abc/g;
```
### 2. RegExp 构造函数（动态拼接规则）
```js
const reg = new RegExp("abc", "g");
// 字符串里 \ 需要转义成 \\
const reg2 = new RegExp("\\d+");
```

## 二、修饰符（写在最后）
| 修饰符 | 作用                                     |
| ------ | ---------------------------------------- |
| `g`    | global 全局匹配，找出所有结果            |
| `i`    | ignoreCase 忽略大小写                    |
| `m`    | multiline 多行模式，`^` `$` 匹配每行首尾 |
| `s`    | dotAll `.` 匹配换行符 `\n`               |
| `u`    | Unicode 支持emoji、生僻汉字              |
| `y`    | sticky 粘性匹配，仅从lastIndex开始       |

示例：全局忽略大小写匹配
```js
const r = /hello/gi;
```

## 三、元字符（核心匹配符号）
### 1. 基础元字符
- `.` 匹配任意单个字符（不含换行，加`s`才匹配换行）
- `\` 转义符：`\.`匹配点、`\*`匹配星号
- `^` 开头；`$` 结尾
```js
/^123$/ // 只能是完整字符串123
```

### 2. 字符类 `[]`
- `[abc]` 匹配 a/b/c 任意一个
- `[0-9]` 数字；`[a-z]`小写；`[A-Z]`大写
- `[^abc]` 取反，匹配**不是**a/b/c的字符
```js
/[0-9a-zA-Z]/ // 数字+大小写字母
```

### 3. 预定义简写类
| 简写 | 等价            | 含义                     |
| ---- | --------------- | ------------------------ |
| `\d` | `[0-9]`         | 数字                     |
| `\D` | `[^0-9]`        | 非数字                   |
| `\w` | `[0-9a-zA-Z_]`  | 单词字符                 |
| `\W` | `[^0-9a-zA-Z_]` | 非单词                   |
| `\s` | `[\t\n\r ]`     | 空白（空格、制表、换行） |
| `\S` | 非空白          |                          |

### 4. 量词（控制出现次数）
- `*` 0次或多次 ≥0
- `+` 1次或多次 ≥1
- `?` 0次或1次
- `{n}` 恰好n次
- `{n,}` 至少n次
- `{n,m}` n~m次
```js
/\d+/      // 连续数字 123、5
/\d{3,6}/  // 3~6位数字
/a?/       // a可有可无
```

#### 贪婪 vs 惰性
默认**贪婪**（尽可能多匹配）；量词后加`?`变惰性（尽可能少）
```js
"1234".match(/\d+/)  // ["1234"] 贪婪
"1234".match(/\d+?/) // ["1"] 惰性
```

### 5. 分组 `()` 与或 `|`
1. `(xxx)` 捕获分组，提取内容
2. `(?:xxx)` 非捕获分组，只分组不提取
3. `a|b` 或，匹配左边或右边
```js
// 匹配 http / https
/https?:\/\//
// 提取手机号区号
/(\d{3})-(\d{8})/
```

### 6. 反向引用
`\1` 代表第1个捕获分组匹配到的内容
```js
/(\w)\1/ // 匹配连续重复字符 aa、bb
```

## 四、正则常用方法
### 1. 字符串方法（最常用）
#### str.match(reg)
匹配结果数组，加`g`返回所有匹配，不加返回完整匹配+分组
```js
"a1b2c3".match(/\d/g) // ["1","2","3"]
```

#### str.search(reg)
返回第一个匹配下标，无匹配返回`-1`
```js
"abc123".search(/\d/) // 3
```

#### str.replace(reg, 替换内容)
替换匹配文本，支持`$1`取分组
```js
// 隐藏手机号中间4位
"13812345678".replace(/(\d{3})\d{4}(\d{4})/, "$1****$2")
// "138****5678"
```

#### str.split(reg)
按正则分割字符串
```js
"1,2;3 4".split(/[,; ]/) // ["1","2","3","4"]
```

### 2. 正则对象方法
#### reg.test(str)
返回布尔值，判断是否能匹配（性能高，判断存在首选）
```js
/\d+/.test("abc123") // true
```

#### reg.exec(str)
一次只匹配一条，配合`g`循环拿全部匹配，携带分组、下标信息
```js
const r = /(\d)/g;
let res;
while(res = r.exec("a1b2")){
  console.log(res[1]) // 1、2
}
```

## 五、经典常用正则示例
1. 手机号（11位，1开头）
```js
/^1[3-9]\d{9}$/
```
2. 邮箱简单校验
```js
/^\w+@\w+\.\w+$/
```
3. 纯数字
```js
/^\d+$/
```
4. 去除所有空格
```js
str.replace(/\s/g, "")
```
5. 提取所有中文
```js
str.match(/[\u4e00-\u9fa5]/g)
```

## 六、常见坑
1. `.` 不匹配换行，多行文本加修饰符`s`
2. 构造函数里反斜杠必须写双斜杠 `\\d`
3. 全局`g`模式下，`test/exec`会记录`lastIndex`，多次调用结果错乱
4. `[]`内部 `. * +` 不需要转义，只有 `\ ^ ] -` 需要处理
5. `^ $` 不加`m`只匹配整个字符串首尾，加`m`匹配每行首尾
