# JavaScript 性能优化完整指南
## 一、变量与内存优化
### 1. 变量声明
1. 优先 `const`，其次 `let`，杜绝 `var`
   - `var` 函数提升、全局污染、变量泄露；`let/const` 块级作用域，引擎优化更好
2. 减少全局变量
   - 全局变量挂载 `window`，查询作用域链更长，访问慢
   - 方案：用 IIFE、模块、类封装变量，局部访问更快
3. 缓存重复访问的对象属性 / DOM
```js
// 差：重复读取 obj.user.info
for(let i=0;i<1000;i++){
  console.log(obj.user.info.name)
}

// 优：缓存
const info = obj.user.info;
for(let i=0;i<1000;i++){
  console.log(info.name)
}
```

### 2. 垃圾回收 GC
JS 自动垃圾回收，频繁创建销毁对象会造成**GC停顿**（页面卡顿）
1. 避免循环内大量创建临时对象
```js
// 差
for(let i=0;i<10000;i++){
  const temp = {x:i}
}
// 优：外部复用对象
const temp = {}
for(let i=0;i<10000;i++){
  temp.x = i
}
```
2. 解除大对象引用：`obj = null` 帮助GC回收
3. 避免闭包长期持有DOM、大数组，导致内存泄漏

## 二、循环性能（高频重点）
1. 四种循环速度对比（由快到慢）
`for(;;)` > `while` > `for...of` > `for...in`
- `for...in` 最慢：遍历原型链、key是字符串，禁止遍历数组使用
- `for...of` 可读性高，但迭代器有开销，超大数组用普通for
2. 循环优化技巧
- 缓存数组长度，不每次读取 `arr.length`
```js
// 差
for(let i=0;i<arr.length;i++){}
// 优
const len = arr.length;
for(let i=0;i<len;i++){}
```
- 倒序循环，减少判断 `i--`
```js
for(let i=arr.length;i--;){}
```
- 循环内少做DOM操作、复杂计算

## 三、DOM 操作优化（前端最大性能瓶颈）
DOM 是浏览器跨线程对象，读写极慢，核心原则：**减少DOM操作次数**
1. 批量渲染，文档碎片 `DocumentFragment`
```js
const frag = document.createDocumentFragment();
for(let i=0;i<100;i++){
  const div = document.createElement('div')
  frag.appendChild(div)
}
// 只一次插入DOM
document.body.appendChild(frag)
```
2. 离线修改：先 `display:none` 修改，再显示，减少重排重绘
3. 缓存DOM节点，不要重复 `querySelector`
```js
// 差
for(let i=0;i<100;i++){
  document.querySelector('#box').innerText = i
}
// 优
const box = document.querySelector('#box')
for(let i=0;i<100;i++){
  box.innerText = i
}
```
4. 避免频繁读取布局属性（offsetTop、clientWidth），会触发强制同步布局

## 四、数组与对象优化
1. 数组
- 尾部增删 `push/pop` 性能极高；头部 `shift/unshift` 会重排所有元素，很慢
  头部操作改用 `reverse()` 或双指针模拟队列
- 预分配数组长度：`new Array(1000)` 比动态扩容更快
- 遍历优先普通for，大数据避免 `forEach`（函数调用有开销）
2. 对象
- 固定结构对象比动态增删属性性能更好（V8引擎隐藏类优化）
- 批量赋值，不要零散新增属性
- 读取键：`.` 访问比 `obj['key']` 略快

## 五、函数与调用优化
1. 减少函数嵌套、避免深层递归（栈溢出+调用开销）
   递归深度大改用循环迭代
2. 防抖、节流控制高频事件：`resize、scroll、mousemove、input`
```js
// 节流示例（限制执行频率）
function throttle(fn, delay=200){
  let flag = false
  return (...args)=>{
    if(!flag){
      flag = true
      fn(...args)
      setTimeout(()=>flag=false, delay)
    }
  }
}
```
3. 避免重复计算，缓存结果（**记忆化 memoize**）
```js
const cache = new Map()
function calc(n){
  if(cache.has(n)) return cache.get(n)
  const res = n ** 10
  cache.set(n, res)
  return res
}
```
4. 箭头函数开销略低于普通function，大量循环可选用

## 六、字符串、正则性能
1. 字符串拼接
- 少量拼接：`+` 简洁
- 大量循环拼接：`Array.join('')` 远快于 `s += str`
```js
// 优
const arr = []
for(let i=0;i<1000;i++) arr.push(i)
const str = arr.join('')
```
2. 正则
- 减少回溯：避免贪婪匹配滥用，长文本用惰性匹配
- 固定正则提取到全局，不要循环内 `new RegExp`

## 七、异步与渲染性能
1. 异步任务分级
- 微任务(Promise) > 宏任务(setTimeout)，大量微任务阻塞渲染
- 耗时计算拆分到 `requestIdleCallback` 空闲执行
2. 耗时计算放到 WebWorker，避免阻塞主线程（JS单线程，计算会卡页面）
3. `requestAnimationFrame` 代替定时器操作DOM动画，对齐浏览器刷新帧率

## 八、运算符与底层提速技巧
1. 位运算替代简单数学运算（轻量、速度快）
- `n >> 1` 等价除以2取整
- `n << 1` 等价乘2
- `n & 1` 判断奇偶
2. 短路运算代替简单if，减少分支判断开销
```js
data && render(data)
```
3. 减少 `if/else` 多层嵌套，大量等值匹配改用 `Map` 映射查找，比switch更快
```js
// 替代多分支
const map = new Map([[1,"新增"],[2,"编辑"],[3,"删除"]])
const text = map.get(type) || "未知"
```

## 九、打包、加载层面性能
1. 代码分割：`import()` 动态导入，减少首屏JS体积
2. Tree Shaking：使用ES Module(`import/export`)，删除未使用代码
3. 压缩混淆：Terser压缩代码，删除注释、缩短变量名
4. 避免引入超大第三方库，优先轻量替代品

## 十、常见性能坑总结
1. 循环内频繁DOM读写、创建临时对象
2. 大量使用 `for...in` 遍历数组
3. 高频事件无防抖节流，疯狂执行逻辑
4. 全局变量泛滥，作用域链查找耗时
5. 递归深度过大、重复无缓存计算
6. 主线程长时间同步大计算，页面卡死
7. 频繁shift/unshift操作数组头部
8. 循环内重复创建正则、DOM查询

## 十一、性能检测工具
1. Chrome DevTools Performance：查看主线程阻塞、GC耗时、重排重绘
2. Console.time() 简易测速
```js
console.time('loop')
for(let i=0;i<100000;i++){}
console.timeEnd('loop') // 打印执行毫秒
```
3. Lighthouse：检测页面加载、JS执行整体性能
