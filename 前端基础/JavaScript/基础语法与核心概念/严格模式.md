# JavaScript 严格模式 `use strict`
## 一、作用
严格模式是 ES5 引入的运行模式，**消除 JS 不合理、不安全语法**，减少诡异隐式转换与静默报错，提升代码规范，同时让引擎优化更快。

## 二、开启方式
### 1. 全局严格模式（写在脚本第一行）
```js
"use strict";
// 全部代码都受严格模式约束
let a = 1;
```
> 注意：前面不能有任何代码、注释外的语句，否则失效。

### 2. 函数内局部严格模式（仅当前函数生效）
```js
function fn() {
  "use strict";
  // 本函数内严格模式
}
// 函数外普通模式
```

## 三、严格模式核心限制（必背考点）
### 1. 禁止未声明变量直接赋值（最常用）
普通模式会自动创建全局变量；严格模式直接抛 `ReferenceError`
```js
"use strict";
num = 10; // 报错：num未定义
```

### 2. 禁止删除变量、函数、参数
普通模式删除变量无提示；严格模式报错
```js
"use strict";
let x = 1;
delete x; // 语法错误
```

### 3. 对象只读属性不可修改、不可删除
```js
"use strict";
const obj = {};
Object.defineProperty(obj, "name", {
  value: "张三",
  writable: false // 只读
});
obj.name = "李四"; // TypeError 报错
delete obj.name;  // 报错
```

### 4. 禁止重复参数名
普通模式允许覆盖，严格模式直接语法报错
```js
"use strict";
function test(a, a) {} // 报错：重复参数
```

### 5. 禁止八进制数字字面量 `0xxx`
普通模式 `010` 视为8进制=8；严格模式报错
```js
"use strict";
let n = 012; // 语法错误
// 如需八进制改用 0o12
let m = 0o12;
```

> 补充历史：老式无字母八进制 012 为什么被淘汰
> 最早 C 语言只设计了 0 代表八进制、0x 代表十六进制，没有二进制、没有 0o。
> 只靠一个 0 区分八进制，出现巨大歧义：
> 09：0 开头但包含 9，不是合法八进制，编译器只能强行降级为十进制 9；
> 011：看着像十进制 11，实际是八进制 9。

严格模式下八进制怎么表示？如下表：

|进制	|英文全称	|词根含义	|前缀|
|--   |--       |--       |--  |
|二进制	|binary	|bi=2	|0b|
|八进制	|octal	|oct=8	|0o|
|十六进制	|hexadecimal	|hex=6，deci=10，6+10=16	|0x|

### 6. `this` 不再自动指向全局 window
普通模式：全局函数 `this === window`
严格模式：全局普通函数 `this = undefined`
```js
"use strict";
function fn() {
  console.log(this); // undefined
  this.num = 1; // 报错，不能给 undefined 赋值
}
fn();
```
> 构造函数不加 new 调用会直接报错，防止污染全局。

### 7. 禁止 `with` 语句
`with` 语法混乱，严格模式直接禁用
```js
"use strict";
with(obj) {} // 语法错误
```

### 8. 给不可扩展对象新增属性报错
```js
"use strict";
const o = {};
Object.preventExtensions(o);
o.age = 18; // TypeError
```

### 9. 禁止在 `eval`、`arguments` 中赋值
```js
"use strict";
eval = 10; // 报错
arguments = [1,2]; // 报错
```

### 10. arguments 不再追踪函数参数变化
普通模式修改 arguments[i] 会同步修改形参；严格模式两者完全隔离
```js
"use strict";
function f(a) {
  a = 100;
  console.log(arguments[0]); // 原始值，不会变成100
}
f(10);
```

## 四、关键字限制
严格模式下，**保留字不能用作变量/函数名**：
`implements, interface, let, package, private, protected, public, static, yield`
```js
"use strict";
let let = 1; // 报错
```

## 五、使用建议
1. 新项目全部开启严格模式，规避大量隐性bug；
2. 框架（Vue/React/Node）默认开启严格模式；
3. Node.js 模块文件默认处于严格模式；
4. 老项目兼容改造建议优先使用**函数内局部严格模式**，避免全局报错。

## 六、优缺点总结
### 优点
1. 杜绝隐式全局变量；
2. 静默失败改为显式报错，方便调试；
3. 限制危险语法（with、八进制、重复参数）；
4. this 行为更可控，减少全局污染；
5. JS引擎可做更多性能优化。

### 缺点
1. 老旧不规范代码会直接报错，改造成本高；
2. 部分老旧语法无法兼容。
