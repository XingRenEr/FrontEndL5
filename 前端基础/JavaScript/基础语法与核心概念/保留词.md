# JavaScript 保留关键字（保留词）
## 一、分类
1. **当前关键字**：现在就不能用作变量/函数名
2. **未来保留字**：ES规范预留，以后可能变成关键字，现在也禁止使用
3. **上下文关键字**：仅特定场景有特殊含义，其余地方可当变量

## 1. 标准关键字（完全不能当标识符）
```
break        do          instanceof    typeof
case         else       let           var
catch        export     new           void
class        extends    return        while
const        finally    super         with
continue     for        switch        yield
debugger     function   this
default      if         throw
delete       import     try
```

## 2. 未来保留字（禁止使用）
### 普通预留
```
enum
```
### 严格模式额外保留词（use strict 里不能用）
```
implements  interface  package  private
protected   public     static
```

## 3. 上下文关键字（仅特定位置有意义，别处可做变量）
只是语法标记，普通变量名可用：
`as` `async` `await` `from` `get` `of` `set` `target`

示例：
```js
// 合法，await 此处只是变量
let await = 123;

async function fn() {
  let p = Promise.resolve();
  await p; // 此处是关键字
}
```

## 4. 不能当标识符的字面量值
`true` `false` `null`
```js
let true = 1; // 报错
```

## 二、使用规则
1. 所有上面的词**不能用作变量名、函数名、类名、对象属性简写**
```js
let class = "a"; // 语法报错
function let(){} // 报错
const obj = { if: 1 } // 报错
```
2. 对象字符串属性不受限制（加引号即可）
```js
const obj = { "class": "测试" }; // 合法
obj.class; // 读取没问题
```
3. 区分大小写：JS关键字全小写，大写如 `Let`、`Class` 可以用
```js
let Let = 10; // 合法
```

## 三、快速记忆高频禁用词
声明类：`var let const class function`
分支循环：`if else switch case for while do break continue`
异常：`try catch throw finally`
异步/生成器：`async await yield`
面向对象：`new super extends import export`
操作：`delete typeof void instanceof return debugger`
预留：`enum public private static`
常量值：`true false null`

## 四、坑点
1. 严格模式下额外多一组保留词（implements/interface等），普通模式不会报错，开启严格模式直接语法错误；
2. `yield` 在普通函数可做变量，在 `function*` 生成器内是关键字；
3. 上下文关键字 `await` 仅在 `async` 函数内受限。
