# JS const 完整详解
## 一、核心定义
`const` ES6 声明**常量**，拥有 `let` 全部特性（块级作用域、暂时性死区、不可重复声明、不挂载 window），**额外限制：声明时必须赋值，且不能重新赋值**。

## 二、核心特性
### 1. 必须初始化，不能只声明不赋值
```js
const a; // 语法错误
const b = 10; // 正确
```

### 2. 禁止重新赋值（栈内存地址不可变）
简单数据类型（数字、字符串、布尔、null、undefined）一旦赋值，无法修改：
```js
const name = "张三";
name = "李四"; // 报错 Assignment to constant variable
```

### 3. 引用类型（对象/数组）仅锁定地址，内部属性可改
`const` 保护的是**变量保存的内存地址**，不是对象内部数据：
```js
// 数组
const arr = [1,2,3];
arr.push(4); // 允许
arr[0] = 99; // 允许
arr = [];    // 报错，不能更换数组地址

// 对象
const user = {age:18};
user.age = 20; // 允许修改属性
user = {};     // 报错
```
如果想要彻底冻结对象，用 `Object.freeze()`：
```js
const obj = Object.freeze({ name:"小明" });
obj.name = "小红"; // 修改无效，严格模式直接报错
```

### 4. 和 let 一致的独有特性
1. **块级作用域**：`{}`、if、for、while 内部隔离，外部访问报错
```js
if(true){
  const num = 100;
}
console.log(num); // 未定义
```
2. **暂时性死区 TDZ**：块内声明前不能访问
```js
console.log(x); // 报错
const x = 5;
```
3. **同一作用域不可重复声明**
```js
const a = 1;
const a = 2; // 报错
```
4. 全局 `const` 不会挂载到 `window`
```js
const test = 666;
console.log(window.test); // undefined
```

## 三、let / const / var 对比
| 特性           | var                     | let                   | const                 |
| -------------- | ----------------------- | --------------------- | --------------------- |
| 作用域         | 函数作用域              | 块级作用域            | 块级作用域            |
| 变量提升       | 可提前访问（undefined） | 存在TDZ，提前访问报错 | 存在TDZ，提前访问报错 |
| 重复声明       | 允许                    | 禁止                  | 禁止                  |
| 全局挂载window | 是                      | 否                    | 否                    |
| 重新赋值       | 可以                    | 可以                  | 不允许                |
| 声明时赋值     | 可选                    | 可选                  | 必须赋值              |

## 四、使用场景
1. **基础固定值**：状态码、接口地址、常量标识
```js
const API_URL = "https://xxx.com/api";
const STATUS_SUCCESS = 200;
```
2. **引用类型容器**：数组、对象（只增删内部数据，不替换整体）
3. 现代开发优先原则：**能使用 const 绝不使用 let**，只有变量后续需要重新赋值时才用 let

## 五、常见坑
1. 误以为 const 对象完全不能修改，仅地址锁定，属性可改；
2. 忘记初始化 `const` 变量直接报语法错误；
3. 暂时性死区陷阱，块内先使用后声明直接报错；
4. 循环中不能用 `const i`，因为每次循环会尝试重新赋值。
