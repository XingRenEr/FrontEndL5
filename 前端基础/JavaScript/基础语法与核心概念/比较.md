# JS 比较运算符全解
## 一、分类
1. 大小比较：`>、<、>=、<=`
2. 相等比较：`==、===、!=、!==`
3. 特殊：`Object.is()`

## 一、大小比较 > < >= <=
规则：
1. 两边都是数字 → 直接比数值
2. 一边数字、一边字符串 → 字符串转数字再比
3. 两边都是字符串 → 按**Unicode编码**逐字符对比
4. `null`转`0`，`undefined`转`NaN`，任何和`NaN`比较都为`false`

```js
10 > 5          // true
"20" > 10       // true ("20"转20)
"100" < "20"    // true 字符串比较：'1'编码 < '2'
null >= 0       // true null→0
undefined > 0   // false undefined→NaN
NaN > 1         // false
```

## 二、相等判断（重点区分 == 与 ===）
### 1. 严格相等 `===` / 严格不等 `!==`
**不做类型转换，类型必须相同 + 值相同才为true**
推荐日常开发全部使用 `===`，避免隐式转换坑。
```js
10 === "10"     // false 类型不同
null === undefined // false
0 === false     // false
"" === false    // false
[] === []       // false 引用地址不同
{} === {}       // false
```

### 2. 宽松相等 `==` / 宽松不等 `!=`
会自动做**隐式类型转换**，规则复杂，尽量少用。
转换核心规则：
1. 数字 vs 字符串：字符串转数字
2. boolean vs 任意：布尔转数字（true→1，false→0）
3. null == undefined → true，除此之外 null/undefined 和其他都不等
4. 对象 == 原始值：对象调用 `valueOf()` / `toString()` 转原始值

```js
10 == "10"      // true
0 == false      // true
"" == false     // true
null == undefined // true
null == 0       // false
[] == ""        // true []转""
[] == 0         // true
```

## 三、null / undefined 特殊对比
```js
null == undefined   // true
null === undefined  // false

// 和数字比较
null >= 0   // true
undefined >= 0 // false
```

## 四、引用类型对比（数组、对象、函数）
`== / ===` 比较**内存地址**，不是内部内容：
```js
const a = [1,2]
const b = [1,2]
a === b // false 两个不同数组

const c = a
c === a // true 同一引用
```

## 五、Object.is() 精准相等
弥补 `===` 两处缺陷：
1. `0 === -0` → true，但 `Object.is(0, -0)` → false
2. `NaN === NaN` → false，但 `Object.is(NaN, NaN)` → true
```js
Object.is(0, -0)    // false
Object.is(NaN, NaN) // true
Object.is(10, "10") // false
```

## 六、运算符优先级
比较运算符（`> < >= <= == === != !==`）优先级**低于算术、高于逻辑运算符**
```js
10 + 5 > 12 && 1 === 1
// 先算 10+5=15，再 15>12=true，1===1=true
// true && true → true
```

## 七、高频踩坑总结
1. 判断相等永远优先 `===`，少用 `==`
2. 字符串和数字混用比较会自动转数字，字符串之间按编码比
3. `NaN` 和任何值（包括自身）`== / ===` 都是 false，判断NaN用 `Number.isNaN()`
4. 对象/数组只比地址，不能直接用 `===` 判断内容是否一致
5. `null == undefined` 是唯一特殊宽松相等组合
