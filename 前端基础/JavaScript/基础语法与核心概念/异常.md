# JS 异常处理完整讲解
## 一、核心语法：try / catch / finally / throw
### 1. 结构
```js
try {
  // 可能报错的代码
} catch (err) {
  // 捕获错误，处理异常
} finally {
  // 无论是否报错，一定会执行
}
```

### 2. throw 主动抛出错误
手动创建异常抛出，中断代码执行，等待 catch 捕获
```js
throw new Error("参数不合法");
// 也可以抛字符串（不推荐）
throw "出错啦";
```

## 二、执行流程
1. 先执行 `try` 块
2. 无报错：跳过 `catch`，直接走 `finally`
3. 有报错：立刻终止 try 剩余代码，进入 `catch`
4. `finally` 永远执行，哪怕 try/catch 里有 return

示例：
```js
function test() {
  try {
    console.log("try执行");
    let a = b; // b未定义，报错
    console.log("这句不会运行");
  } catch (err) {
    console.log("捕获错误：", err.message);
  } finally {
    console.log("finally 必执行");
  }
}
test();
```

## 三、Error 错误对象属性
内置 Error 实例自带属性：
- `message`：错误描述文本
- `name`：错误类型名
- `stack`：调用堆栈（调试定位代码）
```js
try {
  JSON.parse("{invalid}");
} catch (e) {
  console.log(e.name);    // SyntaxError
  console.log(e.message); // JSON.parse解析失败信息
  console.log(e.stack);   // 报错行号堆栈
}
```

## 四、JS 内置错误类型（细分错误）
1. `Error` 基础通用错误（自定义异常用它）
2. `SyntaxError` 语法错误（代码写错，try 捕获不到）
3. `ReferenceError` 引用未定义变量
4. `TypeError` 类型错误（调用非函数、给只读属性赋值）
5. `RangeError` 数值超出有效范围（数组长度负数、递归死循环）
6. `URIError` URI 处理函数参数错误
7. `EvalError` eval 执行异常（极少用）

### 类型判断
```js
try {
  let arr = new Array(-1);
} catch (e) {
  if (e instanceof RangeError) {
    console.log("范围错误");
  } else if (e instanceof TypeError) {
    console.log("类型错误");
  }
}
```

## 五、finally 特殊场景：return 不阻止执行
```js
function fn() {
  try {
    return 100;
  } catch (e) {
    return 200;
  } finally {
    console.log("照样执行");
    // finally 里return会覆盖try/catch的返回值
    return 999;
  }
}
console.log(fn()); // 999
```

## 六、异步代码的异常捕获
### 1. 普通异步回调（try/catch 抓不到）
```js
// 错误示范：捕获不到定时器内部报错
try {
  setTimeout(() => {
    a.b; // 报错，直接抛出全局错误
  }, 100);
} catch (e) {
  console.log("抓不住");
}
// 解决：回调内部单独加 try catch
setTimeout(() => {
  try {
    a.b;
  } catch (e) {
    console.log("捕获成功", e);
  }
}, 100);
```

### 2. Promise 异常
1. `.catch()` 捕获 Promise 错误
```js
new Promise((resolve, reject) => {
  reject("接口失败");
}).catch(err => {
  console.log(err);
});
```
2. async/await 配合 try catch（推荐）
```js
async function request() {
  try {
    const res = await fetch("/api");
  } catch (err) {
    // 网络错误、reject 都会进这里
    console.log("请求异常", err);
  }
}
```

## 七、全局捕获（兜底）
### 1. 同步全局错误监听
```js
window.onerror = function(msg, url, line, col, err) {
  console.log("全局捕获同步错误", msg);
  return true; // 阻止控制台打印默认报错
};
```

### 2. Promise 未捕获拒绝
```js
window.addEventListener("unhandledrejection", e => {
  console.log("Promise未捕获错误", e.reason);
  e.preventDefault(); // 阻止控制台警告
});
```

## 八、自定义错误类（拓展业务错误）
```js
class BusinessError extends Error {
  constructor(msg, code) {
    super(msg);
    this.name = "BusinessError";
    this.code = code; // 自定义错误码
  }
}

// 使用
try {
  throw new BusinessError("账号不存在", 404);
} catch (e) {
  if (e instanceof BusinessError) {
    console.log("业务码：", e.code);
  }
}
```

## 九、常见坑
1. **语法错误 SyntaxError** 无法被 try 捕获，代码解析阶段直接崩溃
2. 定时器、事件回调、异步回调内部报错，外层 try/catch 无效
3. Promise 报错不写 `.catch()` 会触发全局 unhandledrejection
4. finally 中 return 会覆盖 try/catch 的返回值
5. 不要单纯 throw 字符串，建议统一抛 Error 对象，方便获取 stack 堆栈
